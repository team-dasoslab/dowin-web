# 공개 채널 스캔 메모

작성일: 2026-05-20

## 이 문서의 목적

이 문서는 이번 라운드에서 `브런치`, `LinkedIn`, `X`, `Threads`, `Facebook` 공개 영역을 실제로 훑었을 때, 어느 채널에서 유효한 후보 신호가 나왔는지 기록한 메모다.

이 문서의 목적은 아래 세 가지다.

1. 어떤 채널이 실제로 쓸 만했는지 남긴다
2. 어떤 채널은 공개 웹 기준으로 거의 쓸모가 없었는지 남긴다
3. 다음 리서치 라운드의 시간 낭비를 줄인다

## 탐색 범위

- 1차 라운드:
  - 브런치, LinkedIn, X, Threads, Facebook 공개 검색
- 2차 라운드:
  - 동일 채널 기준 추가 10회 이상 검색
  - 브런치 키워드 확장
  - LinkedIn Pulse / 공개 스니펫 확장
  - X / Threads / Facebook 재시도

이번 문서는 최소 10회 이상의 추가 탐색을 반영한 메모다.

## 먼저 결론

이번 라운드에서 실제로 유효한 신호가 나온 채널은 거의 `브런치`였다.

`LinkedIn`은 일부 `Pulse`나 공개 게시물 스니펫이 보조 근거가 되었고,  
`X`, `Threads`, `Facebook`은 공개 검색만으로는 한국인 후보 발굴 효율이 매우 낮았다.

즉 다음 라운드 우선순위는 이렇게 가는 편이 맞다.

1. 브런치
2. LinkedIn 공개 Pulse / 공개 게시물 스니펫
3. 회사 블로그 / 인터뷰 기사
4. X / Threads / Facebook은 공개 검색 보조만

## 채널별 결과

### 1. 브런치

- 결과:
  - 가장 유효했다
- 이유:
  - 작성자 본인의 맥락과 문제의식이 문장으로 길게 남아 있다
  - `대표가 다 본다`, `체계가 없다`, `회의가 비효율적이다`, `액션 아이템 관리가 안 된다` 같은 표현이 나온다
  - `제안하기` 같은 공개 접촉 경로가 남아 있다
- 실제로 건진 것:
  - 셔니
  - Roy
  - iid
  - 알고케어 회의 문화 사례
  - 글쓰는 COO 계열 글

추가 탐색 후 판단:

- 브런치는 아직 더 팔 가치가 있다
- 다만 `실제 인터뷰 요청`으로 바로 갈 만큼 선명한 후보가 빠르게 많아지지는 않았다
- 브런치 안에서도 `운영 일반론`이 아니라 `직접 겪은 장면`만 남겨야 한다

### 2. LinkedIn

- 결과:
  - 보조 근거로는 쓸 만했다
- 이유:
  - 공개 `Pulse` 글이나 검색 스니펫에는 `대표 병목`, `의사결정 기록`, `역할 구분표`, `운영 회의 개입 축소` 같은 표현이 일부 나온다
  - 다만 공개 영역에서 본문 접근이 자주 막히고, 게시물 본문 전체를 안정적으로 보기 어렵다
- 실제로 건진 것:
  - `전능감(全能感)을 내려놓아야 대표는 완성된다`
  - `스타트업 COO – 규칙집 없는 역할`
- 한계:
  - 한국인 실무자 본인 pain보다 번역글/일반론/채용 계열 노이즈가 많았다
  - 추가 탐색 후에도 `지금 바로 DM 보낼 사람`을 브런치보다 잘 주지 못했다

### 3. X

- 결과:
  - 1차 때도 약했고, 추가 탐색 후에도 거의 유효하지 않았다
- 이유:
  - 공개 검색 결과가 계정 소개나 무관한 `follow up` 노이즈로 많이 섞였다
  - 한국 스타트업 실무자 pain를 검색만으로 안정적으로 찾기 어려웠다
  - 로그인/JS 제약으로 맥락 확인이 약했다
- 판단:
  - `X`는 공개 검색만으로는 메인 채널로 쓰기 어렵다
  - 특정 인물을 이미 알고 들어가 확인할 때만 보조 채널로 쓰는 편이 낫다

### 4. Threads

- 결과:
  - 이번 라운드에서는 사실상 유효한 후보를 못 건졌다
- 이유:
  - 공개 검색 노출량 자체가 낮다
  - 한국어 pain 표현을 검색어로 잡아도 실무 후보가 잘 안 나온다
- 판단:
  - Threads는 당장 리드 발굴용으로는 비효율적이다

### 5. Facebook

- 결과:
  - 이번 라운드에서는 거의 완전히 비효율적이었다
- 이유:
  - 검색 결과 대부분이 무관한 페이지, 도움말, 광고 페이지로 흘렀다
  - 한국 스타트업 실무자의 공개 운영 pain를 찾기 어렵다
- 판단:
  - Facebook은 공개 검색 기반 리서치 채널로는 제외하는 편이 낫다

## 이번 라운드의 실무적 의미

이번 조사는 `모든 채널을 다 봐야 한다`는 직감이 꼭 맞지 않다는 걸 보여준다.

지금 단계에서 중요한 건 채널을 넓히는 게 아니라,  
`직접 pain를 길게 남기는 채널`에 시간을 몰아주는 것이다.

그 기준에서 한국어권은 다음이 더 중요하다.

- 브런치
- 회사/개인 블로그
- 인터뷰 기사
- 제한적으로 LinkedIn 공개 글

반대로 아래는 현재 기준으로 우선순위가 낮다.

- X 공개 검색
- Threads 공개 검색
- Facebook 공개 검색

## 추가 탐색 후 바뀐 판단

- `채널을 더 넓히면 후보가 많이 늘어날 것`이라는 기대는 틀렸다
- 공개 웹 기준으로는 브런치와 일부 LinkedIn Pulse를 넘어서는 생산적인 채널이 거의 없었다
- 따라서 다음 단계는 `채널 확장`보다 `브런치 안에서 더 날카로운 pain 표현을 가진 사람을 선별`하거나, `이제는 실제 아웃리치로 전환`하는 쪽이 낫다

## 다음 라운드 권장 방식

1. 브런치에서 먼저 `대표`, `COO`, `CEO Staff`, `운영기획`, `체계`, `회의`, `액션 아이템`, `책임`, `우선순위` 키워드로 더 깊게 판다
2. 브런치 후보를 찾은 뒤에만 LinkedIn 공개 프로필이나 추가 글로 보조 검증한다
3. X / Threads / Facebook은 `후보를 먼저 찾고 나서` 확인하는 용도로만 쓴다

## 현재 기준 한 줄 결론

`이번 공개 채널 스캔에서는 '브런치가 압도적으로 유효했고, LinkedIn은 보조, X·Threads·Facebook 공개 검색은 발굴 채널로 거의 쓸모가 없었다'고 정리하는 편이 맞다.`
