# 직접 pain 표현 기반 한국 인터뷰 타깃 v1

작성일: 2026-05-20

## 이 문서의 목적

이 문서는 `채용공고`나 `운영 역할 존재`가 아니라,  
한국 공개 웹에서 `직접 pain를 말한 사람/팀`만 기준으로 인터뷰 타깃을 다시 뽑은 문서다.

이번 목적은 아래 세 가지다.

1. 우리 제품과 맞닿을 가능성이 있는 한국 후보를 직접 pain 기준으로 고른다
2. 각 후보의 근거를 `직접 표현`, `조직 맥락`, `도구 신호`, `접촉 가능성`으로 나눠 적는다
3. 지금 누구에게 먼저 인터뷰 요청을 보내야 하는지 정한다

주의:

- 이 문서는 `실제 인터뷰 요청 후보`다
- 다만 공개 웹에서 확인 가능한 정보만 썼다
- `우리 고객일 확률이 높다`는 뜻이지 `이미 검증된 고객`은 아니다
- 증거 강도가 약한 후보는 별도로 표시한다

## 평가 기준

이번 문서는 아래 기준을 모두 보려고 했다.

1. `1인칭` 또는 매우 가까운 경험 언어로 pain를 말했는가
2. `회의 전 재정리`, `대표 직접 chase`, `결정 병목`, `체계 부재`, `문서/툴 분산` 중 하나 이상이 드러나는가
3. 스타트업 또는 소규모 팀 맥락이 보이는가
4. 공개 접촉 경로가 있는가

## 증거 강도 기준

- `A`
  - 직접 pain 표현 + 팀/역할 맥락 + 공개 접촉 포인트가 모두 보임
- `B`
  - 직접 pain 표현은 강하지만, 역할/팀 적합도 추정이 조금 들어감
- `C`
  - 문제는 맞지만 우리 ICP와의 연결 고리가 아직 약함

## 이번 라운드 결론

이번 라운드에서는 `셔니`, `글쓰는 COO`, `Roy`가 가장 먼저 볼 만한 후보로 남았다.

이유는 단순하다.

- 한국어로 직접 pain 또는 역할 혼란을 말한다
- 대표/운영/체계/결정 문제를 명시한다
- 공개적으로 인터뷰 요청을 보낼 수 있다

반대로 `툴을 만드는 팀`, `채용공고만 있는 회사`, `문제 일반론만 있는 글`은 이번 문서에서 최대한 제외했다.

추가 10회 이상 탐색 후에도 이 결론은 크게 바뀌지 않았다.

- 새 채널에서 압도적으로 더 좋은 후보가 나오지 않았다
- 오히려 `브런치에서 직접 pain를 길게 쓴 사람`이 계속 가장 유효했다
- 따라서 지금은 후보를 끝없이 늘리기보다 `상위 2~3명에게 실제로 물어보는 단계`로 넘어갈 시점에 가깝다

## P1 후보

### 1. 셔니

- 공개 정체성:
  - `IT 기반 홀딩컴퍼니에서 CEO Staff로 일하고 있습니다`
- 직접 표현:
  - `CEO의 전략적 의사결정을 돕고 조직 운영을 지원하는 포지션`
  - `회사 흐름을 읽고 실행까지 맡는`
  - `지난 1년 동안 투자, 인수합병, 조직 운영, 인사, 마케팅, 세일즈 등 대표가 고민하는 거의 모든 문제를 가장 가까이서 함께 고민하고 해결`
  - `전략팀으로만 한정하기에도 투자, 인사, 세일즈, 운영까지 손대야 하는 범위가 너무 넓었다`
- Dowin과 맞닿는 이유:
  - 대표 옆에서 너무 넓은 운영 문제를 붙들고 있다는 직접 신호
  - `전략 -> 실행 -> 문제 해결` 사이를 오가는 역할이라 1순위 ICP와 가장 가깝다
- 도구/운영 신호:
  - 도구 이름은 직접 안 나오지만, 역할 범위상 분산 정보 정리 pain가 강하게 추정된다
- 접촉 가능성:
  - 브런치 작가 프로필 + `제안하기`

## 현재 판단

이번 v1에서는 `셔니`가 가장 강하다.

이유:

- 현재 CEO Staff로 일하고 있다고 직접 밝힌다
- 대표의 전략/운영/실행 문제를 넓게 다룬다고 직접 쓴다
- 역할 폭이 너무 넓다는 pain가 보인다
- 우리가 찾는 `대표 직속 운영/기획 혼합 역할`과 가장 가깝다

다만 이 후보도 인터뷰 전에는 아래를 확인해야 한다.

1. 회의 전에 상태를 다시 정리하는가
2. follow-up이 흩어지는가
3. owner / due / next action 추적이 어려운가
4. 대표가 직접 다시 chase하는가
5. Notion / Slack / 문서 / 시트가 실제로 분산되어 있는가

추가 10회 이상 탐색 후 판단 보강:

- 더 많은 채널을 봤지만 `셔니`를 넘는 후보는 아직 못 찾았다
- `글쓰는 COO`는 본문 재확인 전까지 보류 가능성이 있다
- 따라서 현재 가장 현실적인 다음 단계는
  - `셔니`
  - `Roy`
  - `iid`
  중 2~3명에게 먼저 가볍게 인터뷰 요청을 보내고, 응답률과 대화 품질을 보는 것이다
