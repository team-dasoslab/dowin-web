# Dowin 대표 직속 실행팀 문제 문장 초안

작성일: 2026-05-09

## 이 문서의 목적

이 문서는 현재 founder-led/대표 직속 운영 세그먼트에서 관찰된 표현을 바탕으로, `대표 직속 실행팀` 세그먼트에 대해 실제로 쓸 수 있는 문제 문장과 포지셔닝 문장 초안을 정리한다.

지금 단계에서는 완벽한 카피보다 아래를 만드는 것이 중요하다.

1. 어떤 pain를 가장 앞에 내세울지 정한다
2. founder/COO/Chief of Staff가 자기 문제라고 느낄 문장을 만든다
3. Dowin 설명이 `또 하나의 협업 툴`로 읽히지 않게 한다

## 먼저 결론

현재 Dowin 기준에서 `대표 직속 실행팀`은 1순위 검증 세그먼트다.

`대표 직속 실행팀`에서 가장 강한 문제는 일이 많은 것이 아니라,  
`우선순위`, `상태`, `결정`, `후속 확인`이 여러 채널에 흩어져 있어서  
대표가 shared picture를 제때 못 보고 다시 직접 chase하게 되는 구조`다.

즉 이 세그먼트에선 아래 문제를 먼저 찔러야 한다.

1. 회의가 실행 회의가 아니라 상태 재구성 회의가 된다
2. 대표가 default approver / follow-up engine으로 다시 돌아온다
3. 팀이 founder energy, shared memory, Slack messages에 기대고 있다

## 관찰에서 반복된 핵심 표현

- `clarity meeting`
- `default approver`
- `decisions and follow-ups start slipping`
- `zero cross-functional visibility`
- `versions of the truth`
- `shared clarity`
- `대표의 의사결정을 문서화`
- `대표의 비전을 구체적인 행동으로 전환`
- `상태/리스크/우선순위 흐릿`
- `결정–실행 간극`
- `단일 사실원천`

## 가장 강한 문제 문장 후보

### 후보 1

`대표가 매주 회의에서 팀의 상태를 처음 확인하게 되는 팀`

왜 강한가:

- 회의 의존
- 상태 파악 지연
- 리더 pain

### 후보 2

`우선순위와 후속 확인이 흩어져 있어서 대표가 계속 직접 chase해야 하는 팀`

왜 강한가:

- founder dependency
- follow-through pain
- 실행 추적 분산

### 후보 3

`Slack, 문서, 회의록은 많은데 이번 주 shared picture가 없는 팀`

왜 강한가:

- scattered context
- tool fatigue
- shared visibility 문제

### 후보 4

`결정은 빠른데 실행 상태가 체계적으로 남지 않아 다시 정리해야 하는 팀`

왜 강한가:

- founder 속도는 빠르지만 운영 구조가 못 받쳐준다는 점을 찌른다
- 국내 맥락과 잘 맞는다

### 후보 5

`팀이 founder의 에너지와 shared memory에 기대고 있어 10명쯤에서 운영이 흔들리는 팀`

왜 강한가:

- 3~15인 팀 가설과 직접 연결된다
- founder-led 팀에 자기 문제처럼 들릴 수 있다

## 리더 언어 버전

대표나 COO에게 직접 말할 때는 아래처럼 더 좋다.

- `지금 팀은 이번 주 상태를 회의 전에 한 번에 보기 어렵지 않나요?`
- `결정은 내려지는데 follow-up이 흩어져 있어서 다시 직접 챙기게 되지 않나요?`
- `중요한 우선순위가 Slack, 문서, 메모에 나뉘어 있어 shared picture가 흐려지지 않나요?`
- `주간 회의가 실행을 앞으로 미는 자리보다 상태를 다시 맞추는 자리가 되지 않나요?`

## 한국어 세일즈/리서치 문장 후보

### 문제 진입 문장

- `대표가 직접 챙기지 않으면 흐려지는 과제가 자주 있나요?`
- `이번 주 핵심 상태를 회의 전에 바로 보기 어려운 편인가요?`
- `대표의 판단은 빠른데 실행 상태가 체계적으로 안 남는다고 느낀 적 있나요?`

### 더 날카로운 버전

- `요즘 팀은 일을 못해서가 아니라, shared picture가 늦게 만들어져서 대표가 다시 직접 개입하게 되는 상태에 가까운가요?`
- `결정은 했는데 누가 언제까지 무엇을 끝내는지가 중간에 흐려지는 순간이 자주 있나요?`

## Dowin 포지셔닝 문장 초안

### 버전 1

`Dowin는 대표 직속 실행팀이 이번 주 우선순위, 실행 상태, 후속 확인을 한곳에서 보고 필요한 개입을 바로 시작할 수 있게 돕는 주간 운영 도구입니다.`

### 버전 2

`Dowin는 회의 때마다 상태를 다시 맞추는 팀이, 목표·액션·진행 상태를 하나의 shared picture로 보고 실행을 앞으로 밀 수 있게 돕는 도구입니다.`

### 버전 3

`Dowin는 대표의 판단과 팀의 실행 사이에 생기는 간극을 줄이기 위해, 우선순위와 follow-up을 주간 운영 리듬 안에서 보이게 만드는 도구입니다.`

## 피해야 할 문장

아래처럼 말하면 이 세그먼트에서 약해질 가능성이 높다.

- `프로젝트 관리 툴입니다`
- `업무 효율을 높여드립니다`
- `협업을 더 잘하게 합니다`
- `회의록과 할 일을 정리해줍니다`

이 표현들은 너무 일반적이거나 `또 하나의 툴`처럼 들릴 수 있다.

## 현재 기준 추천 메시지 방향

가장 유력한 방향은 아래 조합이다.

1. `회의가 상태 재구성 회의가 되어버린 팀`
2. `대표가 직접 chase하는 follow-up 구조`
3. `shared picture 부재`

즉 첫 문제 문장은 이렇게 압축할 수 있다.

`대표 직속 실행팀의 문제는 일이 많은 것이 아니라, 이번 주 shared picture가 늦게 만들어져 대표가 다시 직접 우선순위와 follow-up을 chase하게 되는 구조다.`

## 현재 우선순위 해석

- 이 문서는 현재 1순위 세그먼트에 가장 가까운 문제 문장 초안이다.
- 첫 인터뷰, 첫 세일즈 문장, 첫 랜딩 메시지 실험은 이 문서를 우선 참고하는 편이 맞다.
