# Workspace 도메인 설계

## 1. 개요

**워크스페이스(Workspace)**는 동일한 목표 관리 모임에 참여하는 사람들의 전용 공간이다.  
ADMIN이 워크스페이스를 생성하면, MEMBER가 해당 워크스페이스에 참가한다.

- 현재 구현은 사용자 1명이 여러 워크스페이스에 소속될 수 있다.
- 다만 앱 전역이 모두 다중 workspace-aware로 전환된 것은 아니며, 우선 `active workspace`를 선택하는 기반을 먼저 도입한 상태다.

---

## 2. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | 워크스페이스 생성은 로그인 사용자가 수행하며, 생성자는 해당 워크스페이스의 ADMIN이 된다 |
| BR-2 | 사용자는 여러 워크스페이스에 중복 소속될 수 있지만, 현재 사용 컨텍스트는 1개의 active workspace로 선택된다 |
| BR-3 | ADMIN은 멤버를 강제 퇴출(Kick)할 수 있음 |
| BR-4 | 워크스페이스 삭제는 ADMIN만 가능 (모든 데이터 Cascade Delete) |
| BR-5 | MEMBER는 스스로 워크스페이스를 탈퇴할 수 있음 |
| BR-6 | ADMIN은 권한 이전 또는 워크스페이스 삭제 없이 바로 탈퇴할 수 없음 |
| BR-7 | ADMIN은 다른 멤버에게 관리자 권한을 이전할 수 있음 |
| BR-8 | 워크스페이스 생성/참가/전환 성공 시 `dowin_workspace_id` active workspace 쿠키는 대상 워크스페이스 public uid로 갱신되어야 함 |

---

## 3. 역할(Role) 정의

| 역할 | 설명 |
|------|------|
| `ADMIN` | 워크스페이스 생성자. 멤버 관리 권한 보유 |
| `MEMBER` | 워크스페이스에 참가한 일반 멤버 |

---

## 4. 사용자 흐름 (User Flow)

### 4.1. 워크스페이스 생성 (ADMIN)
```
[온보딩 또는 일반 사용 중]
  └─ "새 워크스페이스 만들기" CTA 클릭
  └─ 워크스페이스 이름 입력
  └─ 생성 완료 → 자동으로 ADMIN 역할로 가입
  └─ 생성한 워크스페이스가 active workspace로 전환
  └─ 대시보드 이동
```

### 4.2. 워크스페이스 참가 (MEMBER)
```
[온보딩 또는 일반 사용 중]
  └─ "참가하기" 버튼 클릭
  └─ 워크스페이스 선택 또는 ID 입력
  └─ 검증 성공 → MEMBER로 자동 가입
  └─ 참가한 워크스페이스가 active workspace로 전환
  └─ 대시보드 이동
```

### 4.2.a. active workspace 전환
```
[워크스페이스 스위처]
  └─ 내가 속한 워크스페이스 목록 조회
  └─ 대상 워크스페이스 선택
  └─ active workspace 쿠키 갱신
  └─ 이후 /api/workspaces/me 및 current-context API가 해당 workspace 기준으로 동작
```

active workspace는 멤버십 role 자체를 바꾸지 않고, 현재 요청 컨텍스트만 선택한다.
다중 워크스페이스 사용자가 한 워크스페이스에서는 `ADMIN`, 다른 워크스페이스에서는 `MEMBER`일 수 있으므로, 조인/전환 직후 쿠키가 갱신되지 않으면 이전 워크스페이스의 role이 현재 화면에 남아 보일 수 있다.
현재 워크스페이스 기준 권한 판정은 `GET /api/workspaces/me.role` 또는 `GET /api/workspaces`의 해당 workspace item `role`을 사용한다.
`GET /api/users/me.role`은 프로필 보조 정보이며 active workspace 권한 판정용으로 사용하지 않는다.

구현 규칙:

- `POST /api/workspaces/join`
- `POST /api/workspaces/join-by-invite`
- `PUT /api/workspaces/current`
- 결제 완료 후 워크스페이스 생성 API

위 경로는 성공 시 `dowin_workspace_id`를 대상 워크스페이스 public uid로 설정해야 한다. 쿠키 속성은 `HttpOnly`, `SameSite=Lax`, `Path=/`를 유지한다.

### 4.3. 멤버 현황 조회 (ADMIN)
```
[설정 > 멤버 관리]
  └─ 전체 멤버 목록 (닉네임, 가입일, 역할 표시)
  └─ 멤버 강제 퇴출 버튼 (확인 다이얼로그)
```

### 4.4. 워크스페이스 탈퇴 (MEMBER)
```
[프로필 > 워크스페이스]
  └─ "워크스페이스 탈퇴" 선택
  └─ MEMBER면 즉시 탈퇴
  └─ ADMIN이면 권한 이전 또는 삭제 먼저 안내
```

### 4.5. 관리자 권한 이전 (ADMIN)
```
[프로필 > 멤버 관리]
  └─ 대상 멤버 선택
  └─ "관리자 권한 이전" 실행
  └─ 대상은 ADMIN, 기존 ADMIN은 MEMBER로 전환
```

### 4.6. 워크스페이스 삭제 (ADMIN)
```
[프로필 > 워크스페이스]
  └─ "워크스페이스 삭제" 선택
  └─ Cascade Delete 경고 확인
  └─ 삭제 후 전체 데이터 제거
```

## 5. 데이터 모델

```typescript
interface Workspace {
  id: string;           // UUID (PK)
  name: string;         // 워크스페이스 이름
  created_at: string;
}

interface WorkspaceMember {
  id: string;
  workspace_id: string;
  user_id: string;
  role: 'ADMIN' | 'MEMBER';
  privacy_level: 'PUBLIC' | 'SUMMARY' | 'PRIVATE'; // Post-MVP
  joined_at: string;
}
```

## 6. API 명세

### GET `/api/workspaces`

> 내가 속한 워크스페이스 목록 조회

---

### PUT `/api/workspaces/current`

> 현재 active 워크스페이스 전환

성공 시 `dowin_workspace_id` 쿠키를 요청한 워크스페이스 public uid로 갱신한다.

---

### GET `/api/workspaces/me`

> 현재 active 워크스페이스 정보 조회

**Response (200 OK)**
```json
{
  "id": "ws_uuid",
  "name": "우리 모임",
  "role": "ADMIN",
  "memberCount": 4
}
```

**Response (404 Not Found)** — 소속 워크스페이스 없음

---

### POST `/api/workspaces`

> ADMIN만 호출 가능 (최초 생성 시)

**Request**
```json
{ "name": "우리 모임" }
```

**Response (201 Created)**
```json
{
  "id": "ws_uuid",
  "name": "우리 모임"
}
```

---

### POST `/api/workspaces/join`

**Request**
```json
{ "workspaceId": "ws_uuid" }
```

**Response (200 OK)**
```json
{ "workspaceId": "ws_uuid", "workspaceName": "우리 모임" }
```

성공 시 참가한 워크스페이스가 active workspace가 되며, `dowin_workspace_id` 쿠키가 해당 워크스페이스 public uid로 갱신된다.

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 404 | 존재하지 않는 워크스페이스 |
| 409 | 같은 워크스페이스에 이미 소속 |

---

### GET `/api/workspaces/:id/members`

**Response (200 OK)**
```json
[
  { "userId": "u1", "nickname": "존", "role": "ADMIN", "joinedAt": "..." },
  { "userId": "u2", "nickname": "민수", "role": "MEMBER", "joinedAt": "..." }
]
```

---

---

### DELETE `/api/workspaces/:id/leave`

> 현재 사용자가 워크스페이스에서 탈퇴

**Response**: `204 No Content`

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 409 | ADMIN은 권한 이전 또는 삭제 없이 탈퇴 불가 |

---

### POST `/api/workspaces/:id/transfer-admin` (ADMIN Only)

> 다른 멤버에게 관리자 권한 이전

**Request**
```json
{ "memberId": 9 }
```

**Response (200 OK)**
```json
{ "message": "관리자 권한을 이전했습니다." }
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 403 | 자기 자신에게 권한 이전 시도 |
| 404 | 대상 멤버를 찾을 수 없음 |

---

### DELETE `/api/workspaces/:id` (ADMIN Only)

> 워크스페이스 삭제

**Response**: `204 No Content`

---

### DELETE `/api/workspaces/:id/members/:memberId` (ADMIN Only)

> 멤버 강제 퇴출

**Response**: `204 No Content`

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 403 | ADMIN 자신을 퇴출 시도 |

---

## 7. 엣지 케이스

| 케이스 | 처리 방식 |
|--------|----------|
| ADMIN이 바로 탈퇴 시도 | 금지. 먼저 관리자 권한 이전 또는 워크스페이스 삭제 필요 |
| ADMIN이 스스로 퇴출 시도 | 금지 |
| 워크스페이스 삭제 시 | 소속 멤버 전원의 Scoreboard/DailyLog Cascade Delete 안내 확인 모달 필요 |
