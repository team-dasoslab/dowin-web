# Workspace 프론트엔드 구현 결과

## 1. 구현 개요

사용자가 워크스페이스에 소속되어 있지 않은 경우를 처리하기 위한 UI와, 초대코드 기반 참가/관리 UI를 구현했습니다. API 연결을 통해 상태를 실시간으로 조회하고 워크스페이스 생성/참가/관리 흐름을 제공합니다.

## 2. 주요 작업 내역

### 2.1. 대시보드 워크스페이스 체크 (`MyDashboardPage`)

- `useGetWorkspacesMe` 훅을 통합하여 실시간 워크스페이스 상태 조회.
- **No Workspace 상태 (404)**: 사용자가 제안한 레이아웃을 바탕으로 "소속된 워크스페이스가 없어요" CTA 화면 구현.
- **로딩 상태 처리**: API 호출 중 스켈레톤 또는 빈 화면 처리 (`null` 반환).

### 2.2. 워크스페이스 생성 페이지 (`/workspace/new`)

- 신규 워크스페이스를 생성할 수 있는 전용 페이지 구현.
- `usePostWorkspaces` 뮤테이션을 사용하여 백엔드와 연동.
- 생성 성공 시 대시보드로 자동 이동.

### 2.3. 워크스페이스 참가 페이지 (`/workspace/join`)

- `usePostWorkspacesJoinByInvite` 뮤테이션으로 `POST /api/workspaces/join-by-invite` 연동.
- 초대코드 입력은 Zod 검증(6~32자) 후 제출.
- 성공 시 `workspaces/me` 쿼리 무효화 후 대시보드로 이동.
- 실패 시 API 에러 메시지를 인라인 에러/토스트로 노출.

### 2.4. 프로필 내 관리자 화면 분리

- `settings/members`는 멤버 조회/퇴출 전용으로 정리.
- `settings/invites`를 신설해 초대코드 생성/복사/활성화 상태 변경 기능 제공.
- `profile` 메인과 `settings/members` 양쪽에 `초대코드 관리` 진입점을 제공.

## 3. 사용된 기술 및 패턴

- **Orval Hooks**: `useGetWorkspacesMe`, `usePostWorkspaces`, `usePostWorkspacesJoinByInvite`, `useGetWorkspacesIdInvites` 등 자동 생성된 TanStack Query 훅 사용.
- **Lucide React**: `Zap`, `Plus`, `ArrowLeft` 아이콘 사용.
- **Tailwind CSS**: 일관된 디자인 시스템 유지를 위한 유틸리티 클래스 활용.

## 4. 검증 완료

- `tsc --noEmit`: 타입 체크 통과.
- `eslint`(변경 파일): 린트 통과.
- `workspaceResponse`의 `status`를 기반으로 한 조건부 렌더링 및 404 분기 처리 정상 동작 확인.
