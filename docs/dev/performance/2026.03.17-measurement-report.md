### 3.1. 요청 조건 고정

- 동일 사용자 세션으로 요청
- 동일 워크스페이스/점수판 데이터셋 사용
- 워밍업 5회 이후 본 측정 30회 수행

### 3.2. 기록 규칙

- Cloudflare 대시보드 또는 동일 출처의 지표만 사용
- CPU Time, Wall Time, Duration은 p50/p95를 모두 기록
- Errors는 총 요청 대비 오류 개수와 비율을 함께 기록

## 6. 해석 메모

- 이번 Run에서 가장 높은 p95 Duration 엔드포인트를 P0 최적화 대상으로 지정한다.
- CPU Time과 Duration이 함께 높은 경우, 애플리케이션 집계 로직 최적화를 우선한다.
- Duration만 높은 경우, DB 조회 범위/쿼리 패턴과 I/O 대기를 우선 점검한다.
