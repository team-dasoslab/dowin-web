# Daily Log 도메인 설계

## 1. 개요

**Daily Log**는 선행지표에 대해 사용자가 특정 날짜에 해당 행동을 **달성했는지(O)** 또는 **몇 회 수행했는지(count)**를 기록하는 가장 핵심적인 인터랙션 단위다.
주간 점수판의 모든 시각화 데이터는 Daily Log를 집계하여 생성된다.

---

## 2. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | `(lead_measure_id, log_date)` 조합은 유일. Upsert 방식으로 처리 |
| BR-2 | 이번 주(월~일) 날짜만 기록/수정 가능. 지난주부터의 과거 날짜는 수정 불가 |
| BR-3 | ARCHIVED 상태의 선행지표에는 Daily Log 추가 불가 |
| BR-4 | `BOOLEAN` 기록은 선택(O) ↔ 미선택(미기록) 토글로 처리 |
| BR-5 | `COUNT` 기록은 `{ count: 1..20 }`만 `PUT`으로 받고, 0회는 `DELETE`로 미기록 처리 |
| BR-6 | `COUNT` 기록은 `count >= dailyTargetCount`인 날짜만 기간 집계에 +1로 반영 |

---

## 3. 주간 집계 정의

| 지표 | 정의 | 계산 |
|------|------|------|
| **주간 달성 수** | 이번 주 달성 날짜 개수 | `BOOLEAN`: `value = true`, `COUNT`: `count >= dailyTargetCount` |
| **주간 목표** | 선행지표의 `target_value` | 예: 7 (매일) 또는 3 (주3회) |
| **달성률(%)** | 주간 달성 수 / 주간 목표 × 100 | |
| **Win 판정** | 달성률 ≥ 80% | 이번 주 "이기고 있음" 뱃지 |

> **주의**: 이번 주는 **월요일 ~ 일요일** 기준 (ISO Week).

---

## 4. 사용자 흐름 (User Flow)

### 4.1. O 토글 기록
```
[대시보드 > 주간 점수판 테이블]
  └─ 각 선행지표 × 요일 셀 클릭
  └─ 미선택 → O (true) → 미선택 순환
  └─ 서버에 Upsert 요청 (낙관적 업데이트)
  └─ 오늘 기준 모든 지표 O 달성 시 Confetti 애니메이션
```

### 4.2. 지난 요일 기록 입력
```
[대시보드 > 날짜 탐색]
  └─ 이번 주 테이블 내에서 지난 요일 열 클릭
  └─ 동일한 O 토글 인터랙션
  └─ 지난주 이전 주차는 조회만 가능하고 수정은 차단
```

## 5. 데이터 모델

```typescript
interface DailyLog {
  id: string;
  lead_measure_id: string;
  log_date: string;       // ISO date 형식 (예: "2026-03-12")
  value: boolean;         // 현재 API 입력은 true만 허용, 삭제 시 미기록으로 복귀
  count: number;          // BOOLEAN은 1, COUNT는 실제 수행 횟수
  created_at: string;
}
```

## 6. API 명세

### PUT `/api/lead-measures/:leadMeasureId/logs/:date`

> 특정 날짜의 기록 생성 또는 수정 (Upsert)

**Path Params**
- `date`: `YYYY-MM-DD` 형식

**Request**
```json
{ "value": true }
```

`COUNT` 지표는 아래 요청만 허용한다.

```json
{ "count": 3 }
```

**Response (200 OK)**
```json
{
  "id": "log_uuid",
  "leadMeasureId": "lm_uuid",
  "logDate": "2026-03-12",
  "value": true,
  "count": 3,
  "achieved": true
}
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 403 | ARCHIVED 선행지표에 기록 시도 |
| 404 | 존재하지 않는 선행지표 |

---

### DELETE `/api/lead-measures/:leadMeasureId/logs/:date`

> 특정 날짜 기록 삭제 ("미기록" 상태로 되돌리기)

**Response**: `204 No Content`

---

### GET `/api/scoreboards/:scoreboardId/logs/weekly`

> 이번 주 전체 선행지표의 일별 기록 조회 (대시보드용)

**Query Params**
- `weekStart`: `YYYY-MM-DD` 형식 (해당 주 월요일, 기본값: 이번 주)

**Response (200 OK)**
```json
{

  "weekEnd": "2026-03-15",

        "2026-03-12": { "value": true, "count": 1, "achieved": true },
```

## 7. UI 가이드라인

### 주간 점수판 테이블 구성

| 선행지표 | 목표 | 월 | 화 | 수 | 목 | 금 | 토 | 일 | 달성 |
|----------|------|----|----|----|----|----|----|----|------|
| 매일 물 2L | 7/7 | ✅ | — | — | ✅ | — | — | — | 2/7 |

- **✅**: `value = true`
- **—**: 미선택 (`null`)
- **오늘 열**: 배경색 강조로 식별 용이하게 처리

### 인터랙션 원칙
- **낙관적 업데이트(Optimistic Update)**: 클릭 즉시 UI 반영 후 서버 동기화
- **실패 시 롤백**: 서버 오류 시 이전 상태로 되돌리고 토스트 에러 노출
- **축하 트리거**: 오늘의 모든 `ACTIVE` 지표가 `true`가 되는 순간 Confetti 발동
