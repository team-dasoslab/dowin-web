# API 응답 컨벤션

## 1. 성공 응답 (Success Response)

### 기본 원칙
- 응답 본문은 항상 JSON.
- 단일 리소스 반환 시 객체, 복수 반환 시 배열을 최상위로 사용.
- 불필요한 wrapper(`{ data: { ... } }`)는 사용하지 않는다.

### HTTP 상태 코드

| 상황 | 상태 코드 |
|------|----------|
| 조회 성공 | `200 OK` |
| 생성 성공 | `201 Created` |
| 수정/액션 성공 (응답 본문 없음) | `204 No Content` |
| 수정/액션 성공 (응답 본문 있음) | `200 OK` |

### 성공 응답 예시
```json
// GET /api/scoreboards/active → 200
{
  "id": "sb_uuid",
  "goalName": "체중을 감량한다",
  "status": "ACTIVE"
}

// GET /api/scoreboards/:id/lead-measures → 200
[
  { "id": "lm_1", "name": "매일 물 2L" },
  { "id": "lm_2", "name": "주 3회 운동" }
]

// POST /api/scoreboards → 201
{
  "id": "sb_new",
  "goalName": "독서 습관을 만든다"
}

// DELETE /api/lead-measures/:id → 204 (본문 없음)
```

## 2. 에러 응답 (Error Response)

### 에러 응답 구조
모든 에러 응답은 동일한 구조를 따른다.

```typescript
interface ErrorResponse {
  error: {
    code: string;    // 머신 리더블 에러 코드 (snake_case)
    message: string; // 사람이 읽을 수 있는 한국어 메시지
    details?: unknown; // 유효성 검사 실패 시 필드별 상세 오류 (선택)
  };
}
```

### 에러 응답 예시
```json
// 401 Unauthorized
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "로그인이 필요합니다."
  }
}

// 403 Forbidden
{
  "error": {
    "code": "FORBIDDEN",
    "message": "해당 리소스에 접근할 권한이 없습니다."
  }
}

// 404 Not Found
{
  "error": {
    "code": "NOT_FOUND",
    "message": "요청한 리소스를 찾을 수 없습니다."
  }
}

// 409 Conflict (비즈니스 규칙 충돌)
{
  "error": {
    "code": "ACTIVE_SCOREBOARD_EXISTS",
    "message": "이미 활성화된 점수판이 있습니다. 기존 점수판을 보관한 후 새로 만들 수 있습니다."
  }
}

// 422 Unprocessable Entity (입력 유효성 오류)
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "입력값이 올바르지 않습니다.",
    "details": {
      "customId": ["ID는 영문/숫자 3~20자만 허용됩니다."],
      "password": ["비밀번호는 8자 이상이어야 합니다."]
    }
  }
}

// 500 Internal Server Error
{
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."
  }
}
```

## 3. 표준 에러 코드 목록

### 공통
| 코드 | HTTP | 설명 |
|------|------|------|
| `UNAUTHORIZED` | 401 | 미인증 또는 세션 만료 |
| `FORBIDDEN` | 403 | 권한 없음 (인증은 됐으나 권한 부족) |
| `NOT_FOUND` | 404 | 리소스 없음 |
| `VALIDATION_ERROR` | 422 | Zod 유효성 검사 실패 |
| `INTERNAL_ERROR` | 500 | 서버 내부 오류 |

### Auth
| 코드 | HTTP | 설명 |
|------|------|------|
| `INVALID_CREDENTIALS` | 401 | ID 또는 비밀번호 불일치 |
| `SESSION_EXPIRED` | 401 | 세션 만료 |
| `WRONG_PASSWORD` | 400 | 현재 비밀번호 불일치 (비밀번호 변경 시) |

### Workspace
| 코드 | HTTP | 설명 |
|------|------|------|
| `INVALID_INVITE_CODE` | 404 | 존재하지 않는 초대 코드 |
| `EXPIRED_INVITE_CODE` | 410 | 만료된 초대 코드 |
| `ALREADY_IN_WORKSPACE` | 409 | 이미 워크스페이스 소속 |
| `CANNOT_REMOVE_LAST_ADMIN` | 403 | 마지막 ADMIN 퇴출 불가 |

### Scoreboard
| 코드 | HTTP | 설명 |
|------|------|------|
| `ACTIVE_SCOREBOARD_EXISTS` | 409 | 이미 활성 점수판 존재 |
| `SCOREBOARD_ARCHIVED` | 403 | 보관된 점수판 수정 시도 |

### Lead Measure
| 코드 | HTTP | 설명 |
|------|------|------|
| `LEAD_MEASURE_ARCHIVED` | 403 | 보관된 지표에 기록 추가 시도 |

### Daily Log
| 코드 | HTTP | 설명 |
|------|------|------|
| `FUTURE_DATE_NOT_ALLOWED` | 400 | 미래 날짜 기록 시도 |
| `PAST_WEEK_LOG_EDIT_NOT_ALLOWED` | 403 | 지난주부터의 과거 기록 수정 시도 |

// src/lib/api-response.ts

import { NextResponse } from "next/server";

type ErrorCode =
  | "UNAUTHORIZED" | "FORBIDDEN" | "NOT_FOUND" | "VALIDATION_ERROR"

  UNAUTHORIZED: 401,
  SESSION_EXPIRED: 401,
  INVALID_CREDENTIALS: 401,
  WRONG_PASSWORD: 400,
  FUTURE_DATE_NOT_ALLOWED: 400,

export const apiError = (code: ErrorCode, details?: unknown) => {
  const status = ERROR_STATUS[code] ?? 400;
  return NextResponse.json(
    { error: { code, message: ERROR_MESSAGES[code], ...(details && { details }) } },

};

export const apiSuccess = <T>(data: T, status = 200) =>
  NextResponse.json(data, { status });

## 5. 인증 미들웨어 컨벤션

```typescript
// src/lib/auth.ts — 세션 검증 유틸

import { cookies } from "next/headers";

const SESSION_COOKIE = "dowin_sid";

export const getSession = async (db: D1Database) => {
  const cookieStore = await cookies();
  const sessionId = cookieStore.get(SESSION_COOKIE)?.value;
  if (!sessionId) return null;

  const session = await db
    .prepare("SELECT * FROM sessions WHERE id = ? AND expires_at > datetime('now')")
    .bind(sessionId)
    .first<{ user_id: string; expires_at: string }>();

  return session ?? null;
};

// API 라우트에서 사용 예시
export async function GET(request: Request, { env }: { env: CloudflareEnv }) {
  const session = await getSession(env.DB);
  if (!session) return apiError("UNAUTHORIZED");

  // ... 이후 로직
}
```

## 6. HTTP 커스텀 Error 클래스

API 로직 깊은 곳에서 비즈니스 규칙 위반이 발생했을 때, 목적에 맞는 HTTP 커스텀 Error 객체를 `throw`하는 패턴을 권장합니다.  
핸들러 최상위의 `try/catch`에서 이를 포획하여 일관된 템플릿(`apiError`)으로 반환합니다.

### 정의 (src/lib/errors.ts)

모든 커스텀 에러는 공통 부모인 `PlatformError`를 상속받으며, HTTP 상태 코드 목적(Bad Request, Unauthorized 등)에 따라 세분화합니다.

```typescript
import { ErrorCode } from "./api-response";

// 공통 부모 에러 클래스
export abstract class PlatformError extends Error {
  public abstract readonly statusCode: number;
  
  constructor(
    public readonly code: ErrorCode,
    public readonly details?: unknown
  ) {
    super(code);
    this.name = this.constructor.name;
    // Error.captureStackTrace(this, this.constructor); // V8 환경(Node) 시
  }
}

// 400 Bad Request
export class BadRequestError extends PlatformError {
  public readonly statusCode = 400;
}

// 401 Unauthorized
export class UnauthorizedError extends PlatformError {
  public readonly statusCode = 401;
}

// 403 Forbidden
export class ForbiddenError extends PlatformError {
  public readonly statusCode = 403;
}

// 404 Not Found
export class NotFoundError extends PlatformError {
  public readonly statusCode = 404;
}

// 409 Conflict (비즈니스 상태 충돌)
export class ConflictError extends PlatformError {
  public readonly statusCode = 409;
}

// 422 Unprocessable Entity (유효성 검사 등)
export class ValidationError extends PlatformError {
  public readonly statusCode = 422;
}
```

### 트라이캐치 래퍼 패턴 (권장)

여러 에러 클래스를 하나의 래퍼에서 안전하게 포획(Catch)합니다.

```typescript
// src/lib/with-error-handler.ts
import { NextResponse } from "next/server";
import { PlatformError } from "./errors";
import { apiError } from "./api-response";

export function withErrorHandler(handler: Function) {
  return async (...args: any[]) => {
    try {
      return await handler(...args);
    } catch (error) {
      // 우리가 정의한 HTTP 목적별 에러인 경우
      if (error instanceof PlatformError) {
        return apiError(error.code, error.details);
      }
      
      // 예상치 못한 시스템 에러 (DB 연결 실패 등)
      console.error("[Unhandled Error]", error);
      return apiError("INTERNAL_ERROR");
    }
  };
}
```
