# Auth 도메인 프론트엔드 연동 결과 (2026.03.14)

## 1. 개요

백엔드에 구축된 인증(Auth) API를 프론트엔드에 통합하여 실제 사용자 인증 흐름(로그인, 로그아웃, 비밀번호 변경)을 구현하고, TanStack Query를 이용한 서버 상태 관리 체계를 구축하였습니다.

## 2. 작업 내용

### 2.1. API 프레임워크 및 인프라 설정

- **TanStack Query 도입**: 전역 상태 관리 및 데이터 패칭을 위해 `src/app/layout.tsx`에 `QueryClientProvider`를 설정.
- **Orval 커스텀 Mutator**: `src/api/mutator.ts`를 작성하여 모든 API 요청 시 `withCredentials: true` 설정을 강제하고, 세션 쿠키(`dowin_sid`) 처리를 자동화. 특히 Orval의 응답 규격 준수를 위해 `data`, `status`, `headers`를 모두 반환하도록 구현하여 연동 안정성 확보.
- **API 클라이언트 생성**: `openapi.yaml` 명세를 기반으로 `yarn gen:api` 명령어를 통해 유형이 안전한(Type-safe) React Query Hook들을 생성.

### 2.2. 컨텍스트 레이어 통합 (Auth Context)

- **MockDataContext 리팩토링**: 기존 Mock 로직을 실제 API 호출로 대체.
  - `login`: `usePostAuthLogin`을 사용하여 인증 후 사용자 프로필 및 `isFirstLogin` 상태를 동기화.
  - `logout`: `usePostAuthLogout`을 호출하여 서버 세션을 종료하고 클라이언트 캐시를 정리한다.
  - `changePassword`: `usePutAuthPassword`를 연동하여 보안 업데이트 기능 추가.
- **사용자 상태 관리**: API 응답 결과를 바탕으로 `User` 타입을 업데이트하고, `isFirstLogin` 여부에 따른 워크플로우 제어 로직 추가.

### 2.3. 페이지 및 UI 연동

- **로그인 페이지 (`src/app/page.tsx`)**:
  - 로그인 성공 후 `/dashboard/my`로 이동하도록 구성.
  - 워크스페이스 조회 결과가 404인 경우 대시보드에서 "소속된 워크스페이스가 없어요" 상태 UI를 즉시 노출.
- **프로필 페이지 (`src/app/profile/page.tsx`)**:
  - 실제 로그아웃 API 연동.
  - "비밀번호 변경" 메뉴에 대화형 프롬프트를 추가하여 실시간 비밀번호 변경 요청 및 피드백(Toast) 구현.

## 3. 결과 확인

- **로그인/로그아웃**: 세션 쿠키 발급 및 삭제, 상태 유지 확인.
- **비밀번호 변경**: 현재/신규 비밀번호 유효성 검사 및 서버 반영 확인.
- **타입 안정성**: Orval 생성 코드와 서비스 로직 간의 인터페이스 불일치 해결 및 빌드 성공 확인.

## 4. 향후 계획

- 현황판(Scoreboard) 및 선행지표(Lead Measures) 도메인 API 연동.
- 사용자 정보 수정을 위한 Profile Update API 추가 연동.
- 에러 핸들링 고도화 (글로벌 에러 바운더리 및 인터셉터 처리).

## 5. 추가 반영 (2026.03.24)

- 로그인/회원가입 통합 화면에서 회원가입 성공 직후 복원코드 확인 단계를 추가했다.
- 가입 성공 시 API 응답의 `recoveryCodes`를 별도 카드 UI로 노출한다.
- 사용자는 복원코드를 복사하고 `계속하기`를 눌러 대시보드로 이동한다.
- 로그인 화면에 `계정 복구` 탭을 추가했다.
  - 복원코드로 계정을 조회하고
  - 새 비밀번호를 즉시 재설정할 수 있다.
- 이후 UX 조정으로 계정 복구를 로그인 탭이 아닌 별도 화면(`/account-recovery`)으로 분리했다.
- 로그인 화면에는 `아이디 혹은 비밀번호를 잃어버리셨나요?` 링크를 노출한다.
