# Dowin Cloudflare WAF 스캐너 Path 차단 기준

## 1. 결론

Dowin는 `Next.js + Cloudflare Workers` 기반 서비스라서 아래 계열 path는 정상 트래픽일 가능성이 매우 낮다.

- PHP/Laravel 노출 파일 탐색
- WordPress/Joomla/Drupal 로그인·관리자·플러그인 경로
- Spring Boot Actuator 같은 운영 엔드포인트
- phpMyAdmin/Adminer/Tomcat Manager 같은 관리 콘솔
- IoT/라우터/Hadoop 계열 취약점 스캔 경로
- Git/AWS credential/환경파일 노출 탐색

반대로 아래처럼 너무 일반적인 path는 로그에 자주 보여도 Dowin에서 일괄 차단 규칙으로 쓰면 안 된다.

- `/api`
- `/admin`
- `/app`
- `/backup`
- `/beta`
- `/cron`
- `/current`
- `/demo`
- `/develop`

이런 값은 위협 인디케이터로는 쓸 수 있어도, path 하나만으로 blanket block 하면 정상 기능이나 향후 운영 경로를 같이 막기 쉽다.

## 2. Cloudflare 규칙 작성 원칙

- `matches` 없이 작성한다.
- 대소문자 변형을 막기 위해 `lower(http.request.uri.path)`를 기준으로 본다.
- exact path는 `in { ... }` 또는 `eq`를 쓴다.
- prefix는 `starts_with()`를 쓴다.
- 중간 포함은 꼭 필요한 경우만 `contains`를 쓴다.
- 규칙 하나를 너무 길게 늘리지 말고 `secret exposure`, `cms/php`, `infra/admin` 정도로 3개 안팎으로 나눈다.

Cloudflare 공식 문서상 custom rule은 `lower()`, `contains`, `starts_with()`, inline list, `in` 연산을 지원하고, Free/Pro 플랜은 regex(`matches`)를 지원하지 않는다.

## 3. 바로 막아도 되는 고신뢰 path 군

### 3.1. Secret / repo / env 노출 탐색

Dowin에서 정상 요청일 이유가 거의 없다.

정확 일치 후보:

- `/.env`
- `/.env.dist`
- `/.env.production`
- `/.env.production.local`
- `/.env.development`
- `/.env.development.local`
- `/.env.prod`
- `/.env.old`
- `/.env.save`
- `/.env.project`
- `/.git`
- `/.git/config`
- `/.aws/credentials`
- `/.aws/config`
- `/phpinfo`
- `/phpinfo.php`
- `/info`
- `/info.php`
- `/_profiler/phpinfo`
- `/frontend_dev.php/$`

prefix / contains 후보:

- `starts_with(path, "/.env.")`
- `contains(path, "/.env")`
- `starts_with(path, "/.git/")`
- `starts_with(path, "/.aws/")`

근거:

- CISA/FBI Androxgh0st advisory가 `/.env`, 여러 `.env*` 변형, `/.git/config`, `/.aws/credentials`, `/.aws/config`, `phpinfo` 계열을 실제 스캔 URI로 제시한다.

### 3.2. PHPUnit / PHP 취약점 탐색

Dowin는 PHP 앱이 아니므로 차단 부작용이 거의 없다.

정확 일치 후보:

- `/vendor/phpunit/phpunit/src/Util/PHP/eval-stdin.php`
- `/xmlrpc.php`
- `/wp-login.php`

prefix 후보:

- `starts_with(path, "/wp-admin")`
- `starts_with(path, "/wp-content/")`
- `starts_with(path, "/wp-includes/")`
- `starts_with(path, "/wp-json/")`
- `starts_with(path, "/phpmyadmin")`
- `starts_with(path, "/pma")`
- `starts_with(path, "/adminer")`

근거:

- CISA는 exposed PHPUnit의 `eval-stdin.php`를 Androxgh0st 관련 악성 요청 URI로 명시한다.
- WPScan은 `xmlrpc.php`가 WordPress의 공격 표면을 넓히며 서버 레벨 차단을 권장한다.
- SANS ISC는 honeypot에서 `phpmyadmin`과 `adminer`가 매우 자주 스캔된다고 설명한다.

### 3.3. Java / infra / 관리 콘솔 탐색

Dowin 현재 스택과 맞지 않는 path라서 정상 요청 가능성이 낮다.

정확 일치 후보:

- `/manager/html`
- `/server-status`
- `/server-info`
- `/actuator`
- `/actuator/env`
- `/actuator/heapdump`
- `/actuator/threaddump`
- `/actuator/prometheus`
- `/metrics`
- `/debug/pprof`
- `/debug/pprof/`
- `/jmx-console`
- `/solr/admin`
- `/v1.16/version`
- `/ws/v1/cluster/apps/new-application`
- `/boaform/admin/formlogin`
- `/login.cgi`
- `/ftptest.cgi`
- `/hnap1`
- `/hnap1/`

prefix 후보:

- `starts_with(path, "/actuator/")`
- `starts_with(path, "/debug/pprof")`
- `starts_with(path, "/solr/")`

근거:

- Spring Boot 공식 문서는 Actuator가 민감 정보를 노출할 수 있으므로 public exposure를 강하게 주의시킨다.
- Wiz는 최근 30일 기준 약 1K 악성 IP가 Spring Boot Actuator endpoint를 스캔했다고 정리했다.
- Prometheus 공식 문서는 `/metrics`와 `/pprof`를 public에 두지 말라고 명시한다.
- Radware의 unsolicited URI 통계에는 `/.env`, `/manager/html`, `/boaform/admin/formLogin`, `/login.cgi`, `/ftptest.cgi`, `/ws/v1/cluster/apps/new-application`, `/v1.16/version`가 상위에 나온다.

## 4. 저신뢰 path 군

아래는 advisory나 honeypot 목록에는 나오지만, path 하나만 보고 block rule로 박는 것은 권장하지 않는다.

- `/api`
- `/admin`
- `/app`
- `/backend`
- `/backup`
- `/beta`
- `/cron`
- `/current`
- `/demo`
- `/develop`
- `/test.php`

이유:

- 일부는 Dowin 현재 경로와 겹치거나, 향후 운영용 내부 페이지/배치/실험 환경에서 다시 쓸 수 있다.
- 이런 값은 path 단독 차단보다 `bot score`, `ASN`, `country`, `method`, `rate limit`, `user-agent`, `known bad IP list`와 조합하는 편이 낫다.

## 5. Dowin용 추천 Cloudflare 표현식

아래 예시는 모두 `matches` 없이 작성했다.

### Rule A. Secret / env / repo 탐색 차단

```text
(
  lower(http.request.uri.path) in {
    "/.env"
    "/.env.dist"
    "/.env.production"
    "/.env.production.local"
    "/.env.development"
    "/.env.development.local"
    "/.env.prod"
    "/.env.old"
    "/.env.save"
    "/.env.project"
    "/.git"
    "/.git/config"
    "/.aws/credentials"
    "/.aws/config"
    "/phpinfo"
    "/phpinfo.php"
    "/info"
    "/info.php"
    "/_profiler/phpinfo"
    "/frontend_dev.php/$"
  }
  or starts_with(lower(http.request.uri.path), "/.env.")
  or contains(lower(http.request.uri.path), "/.env")
  or starts_with(lower(http.request.uri.path), "/.git/")
  or starts_with(lower(http.request.uri.path), "/.aws/")
)
```

권장 액션:

- `Block`

### Rule B. PHP / WordPress / DB admin probe 차단

```text
(
  lower(http.request.uri.path) in {
    "/vendor/phpunit/phpunit/src/util/php/eval-stdin.php"
    "/xmlrpc.php"
    "/wp-login.php"
  }
  or starts_with(lower(http.request.uri.path), "/wp-admin")
  or starts_with(lower(http.request.uri.path), "/wp-content/")
  or starts_with(lower(http.request.uri.path), "/wp-includes/")
  or starts_with(lower(http.request.uri.path), "/wp-json/")
  or starts_with(lower(http.request.uri.path), "/phpmyadmin")
  or starts_with(lower(http.request.uri.path), "/pma")
  or starts_with(lower(http.request.uri.path), "/adminer")
)
```

권장 액션:

- `Block`

### Rule C. Infra / Actuator / router / Hadoop probe 차단

```text
(
  lower(http.request.uri.path) in {
    "/manager/html"
    "/server-status"
    "/server-info"
    "/actuator"
    "/actuator/env"
    "/actuator/heapdump"
    "/actuator/threaddump"
    "/actuator/prometheus"
    "/metrics"
    "/debug/pprof"
    "/debug/pprof/"
    "/jmx-console"
    "/solr/admin"
    "/v1.16/version"
    "/ws/v1/cluster/apps/new-application"
    "/boaform/admin/formlogin"
    "/login.cgi"
    "/ftptest.cgi"
    "/hnap1"
    "/hnap1/"
  }
  or starts_with(lower(http.request.uri.path), "/actuator/")
  or starts_with(lower(http.request.uri.path), "/debug/pprof")
  or starts_with(lower(http.request.uri.path), "/solr/")
)
```

권장 액션:

- `Block`

## 6. 운영 적용 순서

1. 먼저 Cloudflare Security Analytics 또는 origin access log에서 최근 7일 상위 404/403 path를 확인한다.
2. 위 고신뢰 path 군과 겹치는 항목만 1차 룰에 넣는다.
3. 액션은 가능하면 `Block`으로 바로 두되, 불안하면 10~30분 정도 `Log` 또는 `Managed Challenge`로 확인 후 `Block`으로 올린다.
4. `/metrics`, `/server-status`, `/actuator/*`를 내부 모니터링에서 정말 안 쓰는지 한 번 더 확인한다.
5. `/api`, `/admin` 같은 저신뢰 path는 별도 rate limiting 또는 bot-score 조합 룰로 뺀다.

## 7. Dowin 기준 추가 권장사항

- Dowin은 PHP를 쓰지 않으므로 `*.php` 전반을 더 공격적으로 막고 싶을 수 있다.
  - 다만 path extension 전체 차단은 먼저 로그 확인 후 적용하는 편이 안전하다.
- `/.env`, `/.git`, `/vendor/phpunit`, `/wp-*`, `/phpmyadmin`, `/adminer`, `/actuator/*`, `/manager/html`, `/hnap1`, `/boaform/*`는 우선순위가 높다.
- `/api` 계열은 현재 Dowin의 실제 서비스 경로라서 스캐너가 많이 찍는다고 path 단독 차단하면 안 된다.
