# Scoreboard 프론트엔드 구현 결과

## 1. 구현 개요

점수판 프론트는 이제 단순 생성/수정 화면을 넘어서, `내 점수판`과 별도의 `점수판 보관함` 화면에서 활성/보관 상태를 직접 관리할 수 있다.  
보관 후에는 다른 점수판을 다시 활성화하거나 새 점수판을 시작할 수 있고, 보관함에서는 지금까지 만든 점수판 전체를 상시 조회할 수 있다.

## 2. 주요 작업 내역

### 2.1. Setup 화면 상태 전환 보강
- `src/app/(protected)/setup/page.tsx`는 활성 점수판 수정과 새 점수판 생성을 함께 담당한다.
- 수정 모드에서 점수판을 보관하면 훅이 `GET /api/scoreboards/active`의 404를 즉시 반영해 생성 모드로 전환한다.
- 보관 직후 보관 목록에서 재활성화 버튼을 바로 사용할 수 있다.

### 2.2. 점수판 보관함 화면 추가
- `src/app/(protected)/scoreboards/page.tsx`에 점수판 보관함 전용 화면을 추가했다.
- 화면은 `현재 활성 점수판`과 `보관된 점수판` 섹션으로 나뉜다.
- 활성 점수판은 바로 보관할 수 있고, 보관된 점수판은 다시 활성화할 수 있다.
- 보관 카드에는 `활성 기간`을 `startDate - endDate` 형태로 노출한다.

### 2.3. 내비게이션 진입점 추가
- `dashboard/my` 헤더에 `점수판 보관함` 버튼을 추가했다.
- `setup` 화면 헤더에도 같은 진입 버튼을 추가해 작업 중 바로 이동할 수 있다.

### 2.4. 서버 상태 동기화
- `src/app/(protected)/scoreboards/_hooks/useScoreboardArchive.ts`와 `src/app/(protected)/setup/_hooks/useScoreboardSetup.ts`에서
  `getGetScoreboardsActiveQueryKey()`와 `getGetScoreboardsQueryKey()`를 함께 invalidate 한다.
- 활성 점수판 조회가 404일 때는 이전 성공 응답이 캐시에 남아 있어도 화면에서 `activeScoreboard = null`로 처리해 stale UI를 막는다.

### 2.5. Setup 선행지표 횟수 상한 UX 보강
- Setup 선행지표 입력에서 `WEEKLY`는 `targetValue`를 최대 7로 제한한다.
- `MONTHLY`는 점수판 시작일(`startDate`) 기준 해당 월의 최대 일수(28/29/30/31)를 계산해 상한으로 사용한다.
- period 전환 시 기존 값이 상한을 넘으면 즉시 clamp 처리해 저장 직전 오류를 줄인다.
- 월간 입력 옆에 `최대 N회` 힌트를 노출해 사용자 혼란을 낮췄다.

### 2.6. Setup 선행지표 태그 관리 추가
- Setup 화면에서 워크스페이스 공용 태그를 생성하고 선행지표에 최대 3개까지 연결할 수 있다.
- 선택된 태그는 상단 요약 칩으로 노출하고, 하단 선택 목록에서는 숨긴다.
- 태그 이름은 최대 16자이며, 입력 placeholder로 제한을 안내한다.
- 기존 태그 수정/삭제는 노출형 버튼 대신 `...` 메뉴 안에서 처리한다.
- 태그 생성, 수정, 삭제는 낙관적 업데이트로 반영해 개별 작업 중 전역 스피너를 띄우지 않는다.

## 3. 현재 화면 구조

- `src/app/(protected)/dashboard/my/page.tsx`
  - 내 점수판 대시보드
  - 점수판 보관함 진입 버튼 제공
- `src/app/(protected)/setup/page.tsx`
  - 현재 점수판 수정 / 새 점수판 생성
  - 보관 후 재활성화 또는 새 점수판 시작 흐름 지원
- `src/app/(protected)/scoreboards/page.tsx`
  - 전체 점수판 목록 조회
  - 활성 점수판 보관 / 보관 점수판 재활성화

## 4. 현재 제약

- 점수판 보관함은 현재 사용자 자신의 점수판만 보여준다.
- 보관된 점수판 카드는 기간 중심 요약만 제공하며 상세 통계나 히스토리 비교는 아직 없다.
- 점수판 이름/후행지표 외 추가 메타데이터는 보관함 카드에 노출하지 않는다.
