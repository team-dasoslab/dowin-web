# Scoreboard 도메인 설계

## 1. 개요

**점수판(Scoreboard)**은 사용자의 하나의 가중목(Dowin)과 그 성공 기준(후행지표)을 담는 컨테이너다.  
선행지표(Lead Measure)와 일일 기록(Daily Log)은 점수판에 종속된다.

---

## 2. 생명주기 (Lifecycle)

```
[없음] ──(생성)──→ [ACTIVE] ──(보관)──→ [ARCHIVED]
                       │                      │
              선행지표 추가/수정/삭제 가능    재활성화 또는 기록 보관
                       │                      │
                       └──────(재활성화)──────┘
              ※ ACTIVE는 user+workspace당 단 하나
```

---

## 3. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | `user_id + workspace_id` 조합당 `ACTIVE` 점수판은 **최대 1개** |
| BR-2 | ACTIVE 상태를 ARCHIVED로 전환한 후에만 새 점수판 생성 또는 다른 보관 점수판 재활성화 가능 |
| BR-3 | ARCHIVED 점수판은 직접 수정할 수 없고, 다시 사용하려면 먼저 ACTIVE로 재활성화해야 한다 |
| BR-4 | 점수판 생성 시 Dowin 이름(goal_name)과 후행지표(lag_measure)는 필수 |
| BR-5 | 후행지표는 "X에서 Y까지" 형식의 자유 텍스트로 입력 (4DX 가이드 제공) |

---

## 4. 사용자 흐름 (User Flow)

### 4.1. 최초 점수판 생성
```
[대시보드] 점수판 없음 → "새 점수판 만들기" CTA 강조 노출
  └─ Step 1: Dowin 이름 입력
              └─ 가이드: "가장 중요한 목표는 무엇인가요?"
              └─ 힌트: "예: 체중을 감량한다"
  └─ Step 2: 후행지표 입력
              └─ 가이드: "성공을 어떻게 측정할 수 있나요?"
              └─ 힌트: "예: 80kg에서 75kg까지 달성 (3개월 내)"
  └─ Step 3: 시작일 / 목표 기한 입력 (기한은 선택)
  └─ 생성 완료 → 대시보드로 이동 → 선행지표 추가 CTA 노출
```

### 4.2. 점수판 보관(Archive) → 새 점수판 시작
```
[대시보드 > 점수판 메뉴]
  └─ "현재 점수판 마무리하기" 선택
  └─ 확인 모달:
      "점수판을 보관하면 더 이상 기록을 수정할 수 없습니다.
       보관 후 새로운 점수판을 시작할 수 있습니다."
  └─ 확인 → status: ARCHIVED → 기존 보관 점수판 재활성화 또는 새 점수판 생성
```

---

## 5. 데이터 모델

```typescript
interface Scoreboard {
  id: string;
  user_id: string;
  workspace_id: string;
  goal_name: string;      // Dowin 이름 (예: "체중을 감량한다")
  lag_measure: string;    // 후행지표 (예: "80kg에서 75kg까지 달성")
  status: 'ACTIVE' | 'ARCHIVED';
  start_date: string;     // ISO date (예: "2026-03-01")
  end_date: string | null;
  created_at: string;
}
```

## 6. API 명세

### GET `/api/scoreboards/active`

> 현재 활성 점수판 조회 (선행지표 포함)

**Response (200 OK)**
```json
{
  "id": "sb_uuid",
  "goalName": "체중을 감량한다",
  "lagMeasure": "80kg에서 75kg까지 달성",
  "startDate": "2026-03-01",
  "endDate": null,
  "status": "ACTIVE",
  "leadMeasures": [...]
}
```

**Response (404 Not Found)** — 활성 점수판 없음

---

### POST `/api/scoreboards`

**Request**
```json
{
  "goalName": "체중을 감량한다",
  "lagMeasure": "80kg에서 75kg까지 달성",
  "startDate": "2026-03-12",
  "endDate": "2026-06-30"
}
```

**Response (201 Created)** — 생성된 Scoreboard 객체

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 409 | 이미 ACTIVE 점수판 존재 |
| 400 | 필수 필드 누락 |

---

### PUT `/api/scoreboards/:id`

> ACTIVE 상태에서만 수정 가능

**Request** — 수정할 필드만 포함
```json
{ "goalName": "독서 습관을 만든다" }
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 403 | ARCHIVED 점수판 수정 시도 |

---

### POST `/api/scoreboards/:id/archive`

**Response (200 OK)**
```json
{ "id": "sb_uuid", "status": "ARCHIVED" }
```

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 400 | 이미 ARCHIVED 상태 |

---

### POST `/api/scoreboards/:id/reactivate`

> 보관된 점수판 재활성화

**Response (200 OK)** — 재활성화된 Scoreboard 객체

**Error Cases**
| 상태 코드 | 사유 |
|-----------|------|
| 400 | 이미 ACTIVE 상태 |
| 409 | 이미 다른 ACTIVE 점수판 존재 |

---

### GET `/api/scoreboards` (히스토리 조회)

> 보관된 점수판 목록 조회

**Response (200 OK)**
```json
[
  { "id": "sb_old", "goalName": "체중 감량", "status": "ARCHIVED", "startDate": "...", "endDate": "..." }
]
```

## 7. UI 가이드라인

- **Dowin 카드**: 대시보드 최상단에 `[목표명] → [후행지표]` 를 카드로 고정 노출
- **진행 기간**: "시작 N일째" 또는 "마감 D-N" 표시
- **빈 상태 (Empty State)**: 점수판이 없는 경우, 간결하고 동기부여적인 CTA 문구로 생성 유도
  - 예: *"오늘, 당신의 DOWIN를 정의해보세요."*
