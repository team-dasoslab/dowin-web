# Dashboard 도메인 설계

## 1. 개요

**대시보드**는 WIG 서비스의 메인 화면으로, 사용자가 매일 확인하는 "살아있는 점수판"이다.  
별도 데이터 저장 없이, 기존 Scoreboard/LeadMeasure/DailyLog 데이터를 **읽기 전용으로 집계**하여 표시한다.

---

## 2. 뷰 구성

### 2.1. My View (나의 지표 뷰)

사용자 개인의 선택한 기간 달성 현황을 중심으로 표시.

```
┌──────────────────────────────────────┐
│  🎯 WIG 카드                         │
│  "체중을 감량한다"                    │
│  80kg → 75kg / 시작 12일째           │
├──────────────────────────────────────┤
│  📊 선택 기간 현황                    │
│  주간/월간 달성률 + 상태 요약         │
├──────────────────────────────────────┤
│  📋 주간 테이블 또는 월간 집계표       │
│  (선택 보기 단위에 따라 전환)         │
│  선행지표명 아래 태그 읽기 노출        │
└──────────────────────────────────────┘
```

### 2.2. Team View (팀 전체 뷰)

동일 워크스페이스 멤버들의 이번 주 현황을 카드 형태로 나열.

```
┌──────────────────────────────────────┐
│  팀 현황 (이번 주)                   │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │  존    │ │  민수  │ │  지수  │   │
│  │ 🔥Win │ │ 😤Loss│ │ 🔥Win │   │
│  │ 5/7   │ │ 2/7   │ │ 6/7   │   │
│  └────────┘ └────────┘ └────────┘   │
└──────────────────────────────────────┘
```

> 팀 뷰에서는 **이번 주 달성 수/목표**와 **Win/Loss 판정**만 공개.  
> 개별 지표명 및 세부 히스토리는 비공개 (Post-MVP 공개 범위 설정 UI 추가 예정).

## 3. 화면 상태 분기

| 상태 | 표시 내용 |
|------|----------|
| 워크스페이스 미소속 | "초대 코드로 팀에 참가하거나, 새 워크스페이스를 만들어보세요" |
| 점수판 없음 | "오늘, 당신의 WIG를 정의해보세요" + 점수판 생성 CTA |
| 점수판 있음, 선행지표 없음 | 점수판 카드 + "첫 번째 선행지표를 추가해보세요" CTA |
| 점수판 있음, 선행지표 있음 | 전체 대시보드 정상 표시 |

## 4. 핵심 규칙 (Business Rules)

| # | 규칙 |
|---|------|
| BR-1 | `dashboard/my`는 사용자가 선택한 기준 날짜와 보기 단위(`week` / `month`)를 기준으로 집계 |
| BR-2 | Win 판정 기준: 이번 주 전체 달성률 ≥ 80% |
| BR-3 | 팀 뷰는 ACTIVE 점수판이 있는 멤버만 카드 표시 |
| BR-4 | 뷰 전환(My ↔ Team)은 현재 탭 상태를 로컬에서 유지 |
| BR-5 | 점수판 없는 멤버는 팀 뷰에서 "점수판 없음" 상태로 표시 |
| BR-6 | `dashboard` 팀 뷰는 MVP 범위에서 계속 "이번 주" 기준으로 유지 |
| BR-7 | 주간 모드의 날짜 선택은 "주 선택용 앵커"이며, 선택된 날짜가 속한 ISO week(월~일) 7일만 노출 |
| BR-8 | 월간 모드는 선택된 날짜가 속한 달 전체를 집계하며, MVP에서는 월간 집계 중심 UI를 우선 제공 |
| BR-9 | 기준 날짜와 보기 단위는 URL query에 반영하여 새로고침 시 복원 가능해야 함 |
| BR-10 | "오늘" 강조는 실제 오늘 날짜에만 적용하며, 과거/미래 주 탐색 시 선택 기간의 특정 셀을 오늘처럼 오인시키지 않음 |
| BR-11 | 팀 뷰의 사용자별 메모는 현재 선택된 한 사용자 기준으로만 오른쪽 메모 레일에 노출한다 |
| BR-12 | 팀 메모는 같은 워크스페이스 멤버만 조회/생성 가능하며, 완료/삭제는 작성자 또는 워크스페이스 ADMIN만 수행할 수 있다 |
| BR-13 | My View는 선행지표별 태그를 읽기 전용으로 노출하며, 태그 없는 기존 지표도 동일하게 렌더링돼야 한다 |

## 5. API 명세

### My View 데이터 조합

> 현재 `dashboard/my`는 별도 `/api/dashboard/me` 엔드포인트를 사용하지 않는다.

현재 개인 대시보드는 아래 API를 조합해 구성한다.

- `GET /api/workspaces/me`
- `GET /api/scoreboards/active`
- `GET /api/scoreboards/:id/logs/weekly?weekStart=...`
- `GET /api/scoreboards/:id/logs/monthly?monthStart=...`

즉, My View의 기간 탐색 메타와 본문 기록판은 프론트 훅(`useDashboardScoreboard`)에서 조합한다.

### Team Memo APIs (2026-03-25)

> 팀 대시보드 사용자별 메모 조회/생성/완료/삭제

- `GET /api/dashboard/team/memos?targetUserId=...`
  - 현재 로그인 사용자가 속한 워크스페이스를 기준으로, 특정 대상 사용자에 대한 메모 목록을 반환한다.
- `POST /api/dashboard/team/memos`
  - 요청 본문: `targetUserId`, `content`
  - 같은 워크스페이스 멤버를 대상으로 새 메모를 생성한다.
- `PATCH /api/dashboard/team/memos/:memoId/resolve`
  - 요청 본문: `isResolved`
  - 작성자 또는 워크스페이스 ADMIN이 완료 상태를 변경한다.
- `DELETE /api/dashboard/team/memos/:memoId`
  - 작성자 또는 워크스페이스 ADMIN이 메모를 삭제한다.

메모 응답에는 최소 아래 정보가 포함된다.

- 메모 id
- workspaceId
- targetUserId
- 작성자 `userId`, `nickname`, `avatarKey`
- content
- isResolved
- resolvedAt / resolvedByUserId
- createdAt

현재 MVP에서는 메모를 대시보드 집계 응답에 직접 섞지 않고, 별도 메모 API로 분리한다.

## 6. UI 가이드라인

### WIG 카드 (대시보드 최상단 고정)
- 항시 노출: WIG 목표명 + 후행지표 + 경과일 / 남은일  
- 목적: 접속할 때마다 최종 목표 리마인드

### 주간 점수판 테이블
- 모바일: 선행지표명이 긴 경우 truncate 처리 + 상세 진입 가능
- 오늘 날짜 열 강조 (배경색 또는 테두리)
- 사용자별 `메모하기`와 `메모보기`는 점수판 헤더 컨텍스트에서 제공하고, 메모는 오른쪽 보조 레일에 표시

### 기간 선택 UI
- `주간 | 월간` segmented control과 기준 날짜 선택 input 제공
- 주간 모드에서는 날짜 선택기 주변 라벨 또는 helper text로 "선택한 날짜가 포함된 한 주를 보여줌"을 명시
- 이전/다음 기간 이동 버튼과 "오늘로 돌아가기" 액션 제공
- 상단 요약 카드, 라벨, 테이블 제목은 모두 동일한 기준 날짜를 따라야 함

### 월간 집계표
- MVP에서는 선행지표별 `달성 수 / 목표 / 달성률` 중심으로 단순 요약
- 일별 캘린더형 월 뷰는 Post-MVP로 분리

### Win/Loss 뱃지
- Win: 🔥 + "이기는 중" (Indigo / Lime 계열)
- Loss: 😤 + "분발이 필요해" (Muted 계열, 비하 없이 동기부여 톤)

### 뷰 전환 탭
- `나의 지표` | `팀 전체` 탭 — 슬라이드 전환 애니메이션
