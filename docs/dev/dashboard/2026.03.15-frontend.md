# Dashboard 프론트엔드 구현 결과

## 1. 구현 개요

`dashboard/my`와 `dashboard` 팀 뷰를 프로토타입 상태에서 실제 API 기반 화면으로 전환했습니다.  
상세 차트 고도화는 후속 작업 대상으로 남아 있으며, `dashboard/my`의 주간 기록 토글에는 달성 축하 인터랙션을 연결했습니다.

## 2. 주요 작업 내역

### 2.1. My View 실제 데이터 연동
- 워크스페이스 정보는 `useGetWorkspacesMe`로 조회.
- 활성 점수판은 `useGetScoreboardsActive`로 조회.
- 주간 점수판 데이터는 `useGetScoreboardsScoreboardIdLogsWeekly`로 조회.
- 화면에서 기존 mock scoreboard 기반 계산 로직을 제거하고 API 응답 기반으로 교체.

### 2.2. 화면 상태 분기 유지
- 워크스페이스 없음: 생성 CTA 노출.
- 점수판 없음: 새 점수판 만들기 CTA 노출.
- 선행지표 없음: 빈 상태 메시지 유지.
- 정상 상태: WIG 카드 + 주간 점수판 테이블 표시.

### 2.3. 주간 점수판 상호작용
- 각 셀에서 Daily Log 토글 가능.
- 달성률과 `achieved/targetValue` 표시는 주간 로그 응답과 동기화.
- 로딩 중에는 스피너 또는 비활성화 상태를 사용하여 UI 불안정 최소화.

### 2.4. Team View 실제 데이터 연동
- `GET /api/dashboard/team` 계약을 추가하고 Orval 생성 훅으로 조회.
- 팀 카드 영역은 멤버별 점수판 유무, 주간 달성률, 선행지표 요약을 실제 응답으로 렌더링.
- 팀 주간 점수판 영역은 멤버별 선행지표 주간 로그를 읽기 전용 테이블로 렌더링.
- 워크스페이스 없음, 팀원 없음, 점수판 없음 상태를 명시적으로 처리.

### 2.5. 주간/월간 달성률 분리 노출
- My View 상단 카드에서 `이번 주 달성률`과 `이번 달 달성률`을 함께 표시하도록 확장.
- `이번 주 달성률`은 주간 집계 응답을 사용하되 월간 지표를 제외하고 계산.
- `이번 달 달성률`은 신규 `GET /api/scoreboards/:id/logs/monthly` 응답의 `summary.achievementRate`를 사용.
- 카드 라벨에는 월간 응답의 `monthLabel`을 함께 노출.
- 기존 카드/테이블 레이아웃은 유지하고 텍스트 및 수치만 확장.

### 2.7. My View 기간 선택 확장
- `dashboard/my` 상단에 `주간 | 월간` 전환과 날짜 선택 컨트롤을 추가했다.
- 주간 모드에서 날짜 선택은 단일 일자 조회가 아니라, 선택한 날짜가 속한 주(월~일) 전체를 보여주는 앵커로 동작한다.
- 주간 모드의 날짜 input과 URL query는 선택된 주의 시작일(월요일) 기준으로 정규화된다.
- 주간 쿼리는 `weekStart`, 월간 쿼리는 `monthStart`를 사용해 선택 기간 기준으로 조회한다.
- 기존 WIG 카드와 본문 레이아웃은 유지하고, 본문 영역만 `주간 O/X 테이블` 또는 `월간 기록판`으로 전환한다.
- 월간 모드의 본문 목록은 `WEEKLY + MONTHLY` 지표를 모두 표시하고, 상단 월간 요약은 선택한 달의 주간/월간 목표를 함께 반영한다.
- 월간 기록판은 `월~일` 고정 컬럼을 유지한 채 1주차, 2주차처럼 주차별 카드로 나뉘며, 해당 주에 없는 날짜는 빈 셀로 처리한다.
- 월간 기록판의 `달성` 표시는 월 전체 누적값이 아니라 각 주차 카드에 보이는 날짜 범위를 기준으로 계산한다.

### 2.6. 날짜 기준 보정
- `dashboard/my`와 팀 대시보드의 주간 날짜 계산은 클라이언트 KST 기준으로 맞춘다.
- 주간 쿼리 파라미터와 오늘 강조 컬럼도 같은 기준을 사용해 자정 경계에서 화면 오차를 줄였다.

### 2.8. 주간 달성 축하 인터랙션
- `dashboard/my`에서 사용자가 주간 O/X를 토글해 달성 상태가 올라가는 순간을 감지한다.
- 금주 선행지표 중 1개가 처음 100%에 도달하면 기본 강도의 축하 오버레이와 success toast를 노출한다.
- 금주 선행지표 전체가 100%에 도달하면 더 강한 축하 오버레이와 메시지로 승급한다.
- 같은 주 안에서도 체크를 해제했다가 다시 조건을 만족하면 다시 축하를 노출한다.
- 초기 로드나 기간 탐색만으로는 축하가 발생하지 않고, 실제 사용자 토글 직후 전이에만 반응한다.

### 2.9. Free 플랜 히스토리 제한 및 안정성 개선 (2026-04-16)
- **기록 조회 6개월 제한**: 6개월 이전 데이터 조회 시 `403 Forbidden` 에러를 감지하여 해당 영역을 블러 처리하고 안내 오버레이(`HistoryLimitOverlay`)를 노출한다.
- **차트 잠금 처리**: 최근 4주 달성률 차트의 경우, 4주 범위 중 하나라도 6개월 제한 구역에 포함되면 차트를 잠금 상태로 전환한다.
- **네비게이션 연타 방지**: `useTransition`을 활용하여 화살표 및 날짜 선택 클릭 시 즉각적인 UI 잠금(`isPending`)을 수행하고 중복 요청을 차단한다.
- **선제적 이동 차단**: 캘린더 피커와 이전/다음 버튼에 `min` 속성 및 비활성화 상태를 적용하여 6개월 이전으로의 무의미한 이동을 사전에 막는다.
- **주간 보기 예외 처리**: 주간 보기의 경우 해당 주의 일요일(마지막 날)이 제한일 이후라면 조회를 허용하여 6개월 경계 주차의 데이터 누락을 방지한다.

## 3. 구현 방식
- My View는 `useDashboardScoreboard`로 조회/변경 로직 분리.
- Team View는 `useTeamDashboard`로 조회 로직을 분리.
- 페이지 컴포넌트는 상태 분기와 테이블 렌더링에 집중.
- 기존 프로토타입 레이아웃과 스타일은 최대한 유지.

## 4. 검증 결과
- 변경 파일 기준 `yarn eslint` 통과.
- `yarn gen:api` 완료.
- My View와 Team View 모두 실제 API 응답을 사용한 렌더링 구조 확인.
- 현재 저장소 기준 `yarn tsc --noEmit`, `yarn lint`, `yarn test --run` 통과.

## 5. 남은 범위
- 대시보드 상단 종합 요약 카드와 차트 고도화는 Post-MVP 성격의 후속 작업이다.
- 프로필 화면과 `dashboard/my` 헤더는 모두 실제 `GET /api/users/me` 기반으로 닉네임을 표시한다.
