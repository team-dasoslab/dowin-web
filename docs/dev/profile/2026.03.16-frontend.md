# Profile 프론트엔드 구현 결과

## 1. 구현 개요

프로필 화면은 `useGetUsersMe`, `usePutUsersMe`, `useGetWorkspacesMe`를 사용해 실제 서버 데이터와 연결된다.  
현재는 단일 프로필 카드가 아니라 계정 설정, 워크스페이스, 데이터, 알림, 앱 둘러보기, 세션 메뉴를 묶는 허브 역할을 한다.

## 2. 주요 작업 내역

### 2.1. 프로필 조회 연동
- 진입 시 `GET /api/users/me`로 현재 사용자 정보를 조회한다.
- 로딩 중에는 `ProfileSkeleton`을 노출한다.
- 응답 성공 후 닉네임, 아이디, 역할, avatar, 워크스페이스 정보를 화면에 표시한다.

### 2.2. 닉네임 변경 연동
- 닉네임 변경 액션은 `PUT /api/users/me`를 호출한다.
- 성공 시 `getGetUsersMeQueryKey()`와 `getGetDashboardTeamQueryKey()`를 invalidate 한다.
- `GET /api/users/me` 기반 화면들이 다시 동기화되도록 쿼리 캐시를 갱신한다.

### 2.3. 워크스페이스 / 데이터 하위 화면 진입
- ADMIN 사용자는 같은 화면에서 워크스페이스 이름 변경과 멤버 관리(`/profile/members`)에 접근한다.
- 모든 사용자는 CSV 다운로드(`/profile/export`)와 preset avatar 선택(`/profile/avatar`)에 접근한다.
- `새 기능 모아보기`(`/updates`)와 `iPhone 앱 설치 가이드`(`/install-guide`)도 프로필에서 진입한다.

### 2.4. 비밀번호 변경 / 서비스 탈퇴 / 로그아웃
- 비밀번호 변경은 기존 Auth API `PUT /api/auth/password`를 사용한다.
- 서비스 탈퇴는 `DELETE /api/users/me`를 사용한다.
- 서비스 탈퇴는 프로필 화면에서 직접 `prompt`를 띄우지 않고 별도 `/profile/delete-account` 화면에서 처리한다.
- 로그아웃은 `POST /api/auth/logout` 호출 후 Query 캐시를 정리하고 라우팅을 수행한다.
- 앱 전용 알림 전환 이후에는 로그아웃 전에 현재 기기의 FCM 토큰을 disable/remove 한 뒤 세션 로그아웃을 수행해야 한다.

### 2.5. iPhone PWA 설치 가이드
- 프로필 화면에서 `알림 설정`과 분리된 별도 메뉴 그룹으로 `iPhone 앱 설치 가이드`에 진입할 수 있다.
- 실제 라우트는 인증 없이 접근 가능한 공개 경로 `/install-guide`다.
- 가이드는 모바일 기준의 단계형 카드 UI로 구성되며, Safari에서 `홈 화면에 추가`하는 흐름을 6단계로 안내한다.
- 실제 iPhone 캡처 4장은 `public/assets/guide/guide01.png`부터 `guide04.png`까지 정적 자산으로 연결된다.
- 이미지가 아직 없으면 화면이 깨지지 않도록 플레이스홀더를 노출한다.

## 3. 현재 제약

- 닉네임 변경과 워크스페이스 이름 변경 UX는 아직 `prompt` 기반으로 단순 구현되어 있다.
