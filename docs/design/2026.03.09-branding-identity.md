# Dowin 브랜드 아이덴티티: Linear Light Aesthetic

"Dowin"는 **'극단적 정교함(Extreme Precision)'**을 지향하는 프로 도구입니다. 다크 모드 일색인 기존 리니어 스타일을 탈피하여, 고해상도의 하얀 캔버스 위에서 느껴지는 깨끗하고 빠른 사용자 경험을 제공합니다.

## 1. 디자인 키워드 (Design Keywords)

- **High-Contrast Clarity (명료함)**: 백색 배경 위에서 검은 텍스트와 블루 포인트의 완벽한 가시성.
- **Micro-Precision (미세한 선)**: 면 분할이 아닌 1px의 정교한 그레이 라인(Divider) 활용.
- **Layered Depth (겹침의 미학)**: 아주 미세한 그림자(Soft Shadow)를 통한 정보 인지성 확보.
- **Velocity (민첩함)**: 인위적이지 않은 간결하고 빠른 인터랙션.

## 2. 컬러 팔레트 (Color Palette - Premium Light RGBA)

### Core Colors

- **Background**: `rgba(255, 255, 255, 1)` (Pure White)
- **Sub-Background**: `rgba(247, 248, 249, 1)` (Cold Off-white)
- **Primary (Indigo Blue)**: `rgba(58, 100, 199, 1)` (Brand Identity - #3a64c7)
- **Primary Hover**: `rgba(45, 85, 180, 1)`
- **Accent (Lime)**: `rgba(132, 204, 22, 1)` (Momentum Green)

### Typography & Semantics

- **Text Primary**: `rgba(17, 24, 39, 1)`
- **Text Secondary**: `rgba(75, 85, 99, 1)`
- **Text Muted**: `rgba(156, 163, 175, 1)`
- **Success**: `rgba(21, 128, 61, 1)`
- **Danger**: `rgba(225, 29, 72, 1)`

### Borders & Shadows

- **Border**: `rgba(226, 228, 233, 1)`
- **Border Hover**: `rgba(205, 207, 213, 1)`
- **Shadow**: `rgba(0, 0, 0, 0.05)` (Subtle layered shadow)

## 3. 타이포그래피 (Typography)

- **Headings**: `Inter` (Bold)
- **Body**: `Pretendard Variable`
- **Functional Data**: `JetBrains Mono`

## 4. 디자인 메커니즘 (Design Mechanism)

1. **The 1px Rule**: 거의 모든 컴포넌트는 배경색이 아닌 테두리(Border)로 구분됩니다.
2. **Mobile-first & No Hover Shadow**: 터치 인터랙션을 고려하여 hover 시 발생하는 불필요한 그림자나 트랜지션을 지양합니다. 상태 변화는 색상이나 단순한 가시성 변화로 최소화합니다.
3. **Empty Space**: 의도적으로 넓은 여백을 활용하여 작업에 대한 심리적 압박을 줄입니다.

## 5. UI 가이드라인

- **Icons**: Lucide-react (Color: `#626A7D`)
- **Radius**: 6px ~ 8px (샤프한 느낌 강조)
- **Focus States**: 테두리 두께가 아닌 Indigo Blue 색상 변화로 표현.

## 6. Profile Avatar Icon System (2026-03-18)

### 6.1. 목적

- Profile avatar는 사진 업로드의 대체재가 아니라, 제품 안에서 사용자를 빠르게 식별시키는 preset identity token이다.
- 팀 대시보드, 프로필, 향후 공용 user chip에 반복 노출되므로 작은 크기에서도 서로 구분 가능해야 한다.

### 6.2. 스타일 원칙

- 배경이 밝은 환경이 기본이므로, avatar 역시 dark 테마 전제를 두지 않는다.
- `High-Contrast Clarity` 원칙에 따라 외곽 실루엣과 핵심 facial/hair shape가 24px~40px 크기에서도 읽혀야 한다.
- `Micro-Precision` 원칙에 따라 line 위주의 구조를 사용하되, 너무 복잡한 디테일은 배제한다.
- 전체 인상은 귀여운 캐릭터보다 "정교한 프로 도구 안의 정제된 사용자 마커"에 가깝게 유지한다.
- 표현 방향은 Lucide형 선 아이콘이 아니라 `Notion Faces` 계열의 단순 portrait preset에 가깝게 잡는다.
- 단, Notion보다 더 절제된 면 분할과 적은 디테일로 유지해 DOWIN의 light productivity UI와 충돌하지 않게 한다.
- avatar 자체에는 배경색 박스, 원형 프레임, 외곽 border를 두지 않는다. 투명 배경 위 portrait만 남긴다.

### 6.3. 색상 규칙

- 기본 색상 축:
  - Outline/Detail: `rgba(17, 24, 39, 1)` 또는 `#626A7D`
  - Primary Accent: `rgba(58, 100, 199, 1)`
  - Neutral Fill: `rgba(247, 248, 249, 1)`, `rgba(226, 228, 233, 1)`
  - Optional Accent: `rgba(132, 204, 22, 1)`는 소량만 사용
- 과포화 색상, 다색 그라데이션, 무거운 그림자는 사용하지 않는다.
- 얼굴/피부톤은 자연스러운 저채도 범위에서만 사용하고, 헤어/액세서리 색상은 검정, 차콜, 인디고 중심으로 제한한다.

### 6.4. 생성 규칙

- 포맷은 SVG 우선
- 동일 캔버스와 동일 safe area를 사용
- stroke 두께는 세트 내에서 통일
- 각 preset은 아래 요소 중 1~2개만 차이를 두어 식별성을 만든다.
  - head/hair silhouette
  - eye shape or glasses
  - accent ring or badge
  - background fill tone
- 랜덤 생성보다 브랜드 통제가 가능한 수동 curated set을 우선한다.
- 초기 preset은 아래 부품 축을 조합하는 portrait system으로 제작한다.
  - skin tone
  - hair silhouette
  - eye / brow spacing
  - glasses or accessory
  - shoulder top shape
- 작은 크기에서 읽히지 않는 입체 효과, 질감, 세부 음영은 넣지 않는다.

### 6.5. 운영 규칙

- 초기 세트는 8~16종으로 시작한다.
- 파일명/키 규칙은 `avatar-01`, `avatar-02`처럼 단순하고 확장 가능한 패턴을 사용한다.
- 실제 저장값은 이미지 URL이 아니라 `avatarKey`이며, 렌더링 시 asset으로 매핑한다.
