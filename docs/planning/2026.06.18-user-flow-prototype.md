# 팀 멤버 체크인(Inbox) 유저 플로우

Dowin 대시보드 내 **팀 멤버 체크인 위젯(Inbox)**의 전체 유저 플로우(User Flow) 문서입니다. 현재 구현된 프로토타입(챗봇/어시스턴트 기반 UI)을 기준으로 작성되었습니다.

---

## 1. 진입 (Entry)

- **위치**: 대시보드 우측 하단 플로팅 버튼 (Inbox 아이콘)
- **인터랙션**:
  - 버튼 클릭 시 하단에서 위로(Slide-up) 체크인 위젯이 열립니다.
  - 안 읽은 알림이 있을 경우 아이콘 우측 상단에 붉은색 배지로 숫자가 표시됩니다.

---

## 2. 목록 뷰 (Inbox List)

위젯을 열었을 때 가장 먼저 보이는 화면입니다. 이번 주 진행해야 할 액션 아이템들의 상태가 목록 형태로 나열됩니다.

- **표시 정보**:
  - 상단 헤더: `Inbox` 타이틀 및 안 읽은 알림 개수
  - 리스트 아이템: 액션 아이템(목표) 이름, 현재 상태 배지(진행 대기, 조율 진행 중, 조율 완료 등), 수신 시간
- **인터랙션**:
  - 아이템 클릭 시 **상세 뷰(대화창)**로 이동합니다.

---

## 3. 상세 뷰 (대화창 UI)

선택한 액션 아이템에 대해 시스템/리더와 멤버가 메시지를 주고받는 형태의 '챗봇(채팅)' UI입니다. 상태에 따라 다른 화면과 액션을 제공합니다.

### A. 상태: 대기 중 (Pending)

목표에 대해 아직 아무런 기록을 남기지 않은 초기 상태입니다.

1. **시스템 메시지 (수신)**
   - `[시스템]` "이번 주 [목표명] 기록이 아직 없어요. 오늘 가능한 가장 작은 실행을 하나만 업데이트해 주세요."
2. **나의 액션 (하단 버튼)**
   - **빠른 응답 버튼**: `[방금 완료했어요]`, `[오늘 나중에 할게요]`
   - **추가 조율 버튼**: `막힘 있어요`, `목표를 조정할래요`
3. **액션 결과**
   - **빠른 응답 클릭 시**:
     - 시스템에 즉시 상태가 전송됩니다.
     - 1.5초 후 **자동으로 목록 뷰로 복귀**합니다 (연속적인 처리를 위한 UX).
   - **막힘/조율 클릭 시**:
     - 하단에 텍스트 입력창이 나타납니다. ("어떤 어려움이 있나요?" / "어떻게 조율하고 싶으신가요?")
     - 사유를 입력하고 `[전송]` 버튼을 누르면 상태가 업데이트되며 목록 뷰로 복귀합니다.

### B. 상태: 단순 응답 완료 (Done / Later)

'방금 완료했어요' 또는 '나중에 할게요'를 선택한 후, 해당 아이템을 다시 열었을 때의 타임라인입니다.

1. **시스템 메시지 (최초)**: 기록 요청 메시지
2. **나의 응답 (발신)**: "기록을 완료했어요" 또는 "나중에 하기로 했어요"
3. **시스템 피드백 (수신)**: "멋진 실행입니다! 계속 이렇게 나아가요." 등 시스템의 자동 격려 메시지

### C. 상태: 막힘/조율 요청 완료 (Blocked / Adjust)

텍스트를 입력해 리더에게 조율을 요청한 상태입니다. 리더의 답변을 기다립니다.

1. **시스템 메시지 (최초)**: 기록 요청 메시지
2. **나의 응답 (발신)**: 입력한 사유 텍스트 (예: "일정상 무리일 것 같아 목표 하향 조율이 필요합니다.")
3. **시스템 피드백 (수신)**: "막힘 상태를 리더에게 전달했어요." 등 접수 완료 메시지

### D. 상태: 조율 진행 중 (Leader Comment / Proposal)

리더가 나의 요청을 확인하고 피드백이나 새로운 목표 수치를 제안한 상태입니다.

1. **타임라인 히스토리**: 최초 요청 ➔ 나의 조율 요청 텍스트가 상단에 표시됩니다.
2. **리더의 메시지 (수신)**
   - `[리더 이름]`의 말풍선: "제가 영업팀 리소스 붙여드릴 테니 너무 부담 갖지 마시고 3건으로 조정해서 진행해 보시죠!"
3. **리더의 제안 카드 (Optional)**
   - 메시지 하단에 구체적인 변경 사항 요약 카드 (예: `목표 횟수 하향 (5건 ➔ 3건)`)
4. **나의 액션 (하단 버튼)**
   - `[기존 목표 유지]` (제안 거절)
   - `[제안 수락하기]` 또는 `[수정사항 반영]`
5. **액션 결과**
   - 선택 시 상태가 업데이트되며 목록 뷰로 복귀합니다.

### E. 상태: 조율 완료 (Adjusted)

리더의 제안을 수락했거나 기존 목표 유지를 확정한 최종 상태입니다.
하단 액션 버튼이 사라지고, 위 D단계까지의 모든 대화 히스토리가 타임라인으로 고정되어 표시됩니다.

---

## 4. UI/UX 핵심 설계 원칙

- **대화형(Chat) 타임라인**: 모든 알림과 액션은 단순 폼(Form)이 아닌, 시스템/리더와 대화를 나누는 타임라인(말풍선) 형태로 누적됩니다.
- **연속 처리(Inbox Zero) 최적화**: 액션(완료, 나중에 하기 등)을 취하면 즉시 목록으로 돌아가 다음 알림을 처리할 수 있게 하여 조작 피로도를 줄입니다.
- **Toss 스타일의 명확한 위계**: 메인 액션(완료)은 눈에 띄는 파란색 큰 버튼으로, 부가 액션(막힘/조율)은 텍스트 링크로 분리하여 유저가 고민 없이 직관적으로 누를 수 있게 디자인했습니다.
