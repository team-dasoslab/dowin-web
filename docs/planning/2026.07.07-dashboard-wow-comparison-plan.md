# 대시보드 전주 대비(WoW) 증감 표시

## 1. 배경 및 목적 (Background & Context)

사용자가 현재 자신의 진행 상태와 팀원들의 진행 상태를 볼 때, "이번 주에 잘하고 있는지"를 판단하기 위한 기준점이 부족합니다.
지난주 달성률 대비 이번 주 달성률의 증감(+, -)을 표시함으로써, 성장에 대한 동기부여를 강화하고 직관적인 피드백을 제공하고자 합니다.

## 2. 해결하고자 하는 문제 (Problem Statement)

- 현재 대시보드(My, Team)에서는 이번 주/이번 달의 절대적인 달성률만 노출됩니다.
- 사용자는 자신이 지난주보다 나아졌는지, 혹은 뒤처지고 있는지를 한눈에 파악하기 어렵습니다.

## 3. 기획 상세 (Action Items & Scope)

### 3.1. 백엔드 및 API 명세 변경

프론트엔드에서 계산 로직을 최소화하기 위해, 백엔드에서 이전 주(last week)의 달성률 혹은 증감값을 응답에 포함합니다.

**수정 대상 API:**

1. `TeamDashboardResponse`의 `TeamDashboardMember`
   - `lastWeeklyAchievementRate: number | null` 추가 (지난주 점수판이 없었거나 데이터가 없으면 null)
2. `MyDashboardResponse` 또는 `WeeklyLogsResponse`
   - `lastWeeklyAchievementRate: number | null` 추가

**데이터 산출 로직:**

- 조회하는 주(weekStart) 기준, 바로 이전 주(weekStart - 7일)의 달성률을 계산.
- 지난주에 활성 점수판이 없었거나, 선행지표가 0개였다면 비교가 불가하므로 `null` 반환.

### 3.2. 프론트엔드 UI 변경

My 뷰와 Team 뷰 모두 **각 선행지표(액션아이템)**별로 전주 대비 달성 횟수 증감을 표시합니다.
(참고: 주간 달성률(%) 전체 집계 영역에는 증감을 표시하지 않기로 결정되었습니다.)

**My View (나의 지표 뷰)**

- 개별 선행지표 명칭(예: test101-2) 바로 앞에 해당 지표의 달성 횟수 증감 표시
- 예시: `▲ 1 test101-2` (상승), `− test101-2` (동일), `▼ 1 test101-2` (하락)

**Team View (팀 전체 뷰)**

- 팀원의 주간 점수판 테이블(체크인 뷰)에서 개별 선행지표 명칭 앞에 해당 지표의 달성 횟수 증감 표시

**스타일링 가이드:**

- 양수(+): 긍정적 색상 (초록색 계열, `▲ 1`)
- 변동 없음(0): 무채색 (회색 계열, `-`)
- 음수(-): 주의 색상 (빨간색 계열, `▼ 1`)

## 4. MVP vs Post-MVP

**MVP (이번 작업 대상)**

- 백엔드 API 응답에 `lastWeeklyAchievementRate` 추가
- My View 메인 주간 달성률 옆에 증감 UI 노출
- Team View 멤버 카드에 증감 UI 노출

**Post-MVP**

- 월간 모드(Month View)에서의 MoM(전월 대비) 비교 기능
- 증감률에 따른 동기부여 메시지(Toast/Memo) 자동 생성

## 5. 성공 기준 (Success Criteria)

- API 스펙 갱신 후 하위 호환성 문제 없이 프론트엔드가 이를 렌더링할 수 있다.
- 사용자가 전주 대비 달성률의 변화를 시각적으로 즉각 파악할 수 있다.
