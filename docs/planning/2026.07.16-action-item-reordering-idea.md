# 액션 아이템 수동 정렬 (위/아래 버튼)

## 1. 문제 및 배경
- **문제**: 사용자들은 액션 아이템의 우선순위를 지정하고 싶어 하지만, 명시적인 "우선순위" 필드(예: 상/중/하)를 추가하면 인지적 부담이 늘어나고 생성 UI가 복잡해집니다. 드래그 앤 드롭(DnD)은 직관적이지만 모든 기기에서 완벽하게 구현하기 까다로울 수 있습니다.
- **도입 필요성**: 액션 아이템이 쌓일수록 우선순위가 없으면 가장 중요한 작업에 집중하기 어렵습니다.
- **범위**: Post-MVP (아이디어 단계)
- **영향받는 도메인**: `dashboard` (프론트엔드), `action-item` (백엔드/DB)

## 2. 제안하는 해결책 (위/아래 버튼)
복잡한 DnD나 우선순위 태그 대신, 각 액션 아이템 옆에 단순한 **[↑] (위로)** 및 **[↓] (아래로)** 버튼을 제공합니다.
- [↑] 클릭 시: 바로 위 아이템과 위치를 바꿉니다.
- [↓] 클릭 시: 바로 아래 아이템과 위치를 바꿉니다.

### 기대 효과
- **장점**: 
  - 아이템 생성 시 인지적 부담이 전혀 없습니다.
  - 완전한 드래그 앤 드롭에 비해 프론트엔드 구현이 매우 단순하고 견고합니다.
  - 데스크탑과 모바일 웹 모두에서 일관되게 잘 동작합니다.
- **단점**:
  - 긴 목록의 맨 아래에 있는 아이템을 맨 위로 옮기려면 여러 번 클릭해야 합니다.

## 3. 기술적 타당성 (백엔드 & DB)
수동 정렬을 지원하려면 데이터베이스에 순서를 저장할 방법이 필요합니다.
- **접근법 1: `order_index` (Integer 교환)**
  - 액션 아이템 테이블에 정수형 `order_index` 컬럼을 추가합니다.
  - [↑] 클릭 시, 타겟 아이템과 바로 위 아이템의 `order_index` 값을 서로 교환합니다. 정확히 2개의 row만 업데이트하면 됩니다.
- **접근법 2: Fractional Indexing (분수 인덱싱)**
  - 실수형(float) 또는 문자열 `position` 컬럼을 추가합니다.
  - [↑] 클릭 시, 위쪽 아이템과 그 위쪽 아이템 사이의 새로운 중간값을 계산합니다. 1개의 row만 업데이트합니다.
- **권장 사항**: 한 번에 한 칸씩만 이동하는 위/아래 버튼의 UX 특성상, **접근법 1 (Integer 교환)**이 훨씬 단순하고 예측 가능합니다.

## 4. 액션 아이템 (향후 구현 시)
이 기능을 진행하기로 결정할 경우:
- [ ] **백엔드**: D1 데이터베이스의 액션 아이템 테이블에 `order_index` 컬럼 추가.
- [ ] **백엔드**: 순서 교환 로직을 처리할 API 엔드포인트(예: `PATCH /action-items/:id/reorder`) 생성.
- [ ] **프론트엔드**: 액션 아이템 리스트 컴포넌트에 위/아래(↑, ↓) UI 버튼 추가.
- [ ] **프론트엔드**: TanStack Query 뮤테이션을 연결하여 순서 변경 API를 호출하고 리스트를 낙관적 업데이트(Optimistic Update) 처리.

## 5. 남은 질문들 (Open Questions)
