# `dashboard/my` 실행 가이드 툴팁 기획안

## 1. 배경 및 맥락 (Background & Context)

- 현재 DOWIN는 점수판, 주간/월간 기록판, 팀 대시보드, 푸시 리마인드까지 갖추고 있다.
- 하지만 사용자가 특정 선행지표를 지금 유지해야 하는지, 횟수를 낮춰야 하는지, 아예 다른 행동으로 바꿔야 하는지까지 번역해 주는 레이어는 없다.
- 그 결과 기록이 밀릴수록 대시보드를 덜 열게 되고, 주 후반에 몰아서 기록하거나 그대로 놓치는 흐름이 생길 수 있다.
- 이 기능은 회고처럼 "끝나고 정리하는 도구"가 아니라, 특정 선행지표의 설정과 실행 방식을 조정하게 돕는 "진행 중 코칭"으로 정의한다.

## 2. 우선순위 및 당위성 (Priority & Rationale)

- 현재 제품 우선순위는 유료화보다 `무료 가치 강화`와 `반복 사용 신호 확보`다.
- 실행 가이드는 기록 데이터를 더 많이 쌓게 만드는 것이 아니라, 기록이 잘 안 붙는 선행지표를 더 현실적인 행동으로 조정하게 만드는 기능이다.
- 개인 사용감에서 먼저 검증하더라도, 이후 팀 단위로 확장할 때 `누가 위험한가`가 아니라 `어떤 실행 안내가 필요한가`라는 팀 친화적 관점으로 이어질 수 있다.

## 3. 제품 원칙 (Product Principles)

- 이 기능은 `위험 판정기`가 아니라 `실행 코치`여야 한다.
- 사용자를 평가하거나 압박하지 않고, 조정 가능한 다음 행동만 보여준다.
- 경고 문구, 실패 예측, 낙인형 상태 배지는 금지한다.
- 결과 해석보다 행동 조정 제안에 초점을 둔다.
- 잘하고 있는 지표는 굳이 칭찬 문구를 붙이지 않고 침묵한다.

## 4. MVP 범위 (MVP Scope)

### 4.1. 위치

- `dashboard/my` 주간 점수판의 각 선행지표 이름 옆에 작은 아이콘 트리거를 노출한다.
- 아이콘을 누르면 해당 선행지표에 대한 실행 가이드 툴팁이 열린다.
- 월간 모드에서는 노출하지 않고, 우선 `week` 보기에서만 적용한다.

### 4.2. 노출 대상

- 활성 점수판이 있고 `selectedView === "week"`인 개인 대시보드
- `WEEKLY` 또는 `DAILY` 성격으로 해석되는 주간 선행지표
- 생성한 지 2주가 지난 선행지표만 대상
- 최근 2주 기준으로 조정 또는 변경 제안이 필요한 지표만 노출
- 최근 2주 모두 50% 이상인 지표는 아무 것도 노출하지 않는다.

### 4.3. 입력값

- 선행지표 생성일
- 직전 주 달성 횟수
- 이번 주 달성 횟수
- 선행지표 목표 횟수
- 선행지표 주기(`WEEKLY`, `DAILY`, `MONTHLY`)

### 4.4. 출력값

- 작은 아이콘 트리거 1개
- 클릭 시 짧은 제안 문구 1개

예시:

- "2주 연속 50% 미만이에요. 이 선행지표는 횟수를 조금 낮춰보세요."
- "2주 연속 기록이 없어요. 이 선행지표는 다른 행동으로 바꿔보세요."

## 5. 톤 및 카피 규칙 (Tone & Copy Rules)

### 금지 표현

- `위험`
- `실패`
- `불가`
- `늦었다`
- 강한 경고색 중심 피드백
- "왜 아직 안 했나요"처럼 사용자를 추궁하는 표현

### 권장 표현

- `횟수를 낮춰보세요`
- `다른 행동으로 바꿔보세요`
- `이 선행지표는`
- `2주 연속`
- `조금`

카피 원칙:

- 상태 평가보다 조정 가능성을 먼저 보여준다.
- 사용자의 부족함보다 바꿀 수 있는 선택지를 먼저 보여준다.
- "이 지표는 안 된다" 같은 단정 대신, 횟수 조정 또는 행동 교체라는 구체적 액션을 제안한다.

## 6. 판정 규칙 초안 (Decision Rules)

- 복잡한 예측 모델 없이 최근 2주의 단순 달성률 비교만 사용한다.
- 가능한 한 주간 로그 응답 2주치만으로 계산한다.

예시 분기:

1. 생성 후 2주가 지나지 않음
   - 아무 것도 노출하지 않음
2. 최근 2주 연속 `achieved === 0`
   - `선행지표 변경 제안`
3. 최근 2주 연속 `achievementRate < 50`
   - `횟수 조정 제안`
4. 최근 2주 연속 `achievementRate >= 50`
   - 아무 것도 노출하지 않음

즉, `좋다/나쁘다`를 계속 평가하지 않고, 조정이 필요한 케이스만 조용히 알려준다.

## 7. MVP 제외 범위 (Non-Goals)

- 팀원 비교 또는 팀 요약 경고
- 푸시 알림 자동 발송 연동
- AI 생성형 추천 문구
- 사용자가 규칙을 직접 커스터마이징하는 기능
- 월간 모드용 별도 가이드
- 회고 작성/저장 기능
- `좋은 흐름` 칭찬 노출
- 상단 총괄 카드형 실행 가이드

## 8. Post-MVP 후보 (Post-MVP)

- 팀 대시보드용 운영자 관점 실행 가이드
- 주간 종료 시 회고 카드와 연결
- 가이드 노출 시점 기반 이벤트 로그 수집 및 메시지 효과 비교
- 푸시 리마인드와 가이드 문구 연동
- 칭찬형 가이드를 넣을지 여부 재검토

## 9. 영향 도메인 (Domain Impact)

- `dashboard/my` 주간 선행지표 행 UI
- 주간 로그 2주 범위 집계 결과의 해석 로직
- 추후 analytics 이벤트 정의

## 10. 성공 및 실패 기준 (Success & Failure Criteria)

### 성공

- 사용자가 왜 이 아이콘이 떴는지 눌러보면 바로 이해할 수 있다.
- 경고나 죄책감 없이 `횟수를 낮출지`, `행동을 바꿀지`가 선명해진다.
- 기존 점수판/기록 UX를 가리지 않고 조용히 보조한다.

### 실패

- 아이콘이 너무 많아 점수판을 어지럽힌다.
- 문구가 추상적이라 실제 행동 조정으로 이어지지 않는다.
- 생성 직후 지표에도 가이드가 떠서 불필요한 압박으로 느껴진다.
- 툴팁 위치나 표시 방식이 표를 가리거나 깨뜨린다.

## 11. 다음 액션 (Next Actions)

- `GET /scoreboards/{scoreboardId}/logs/weekly` 응답에 지표별 guide 필드를 포함할지 확정한다.
- 최근 2주 판정 규칙과 생성일 2주 경과 규칙을 backend에서 계산하도록 정리한다.
- 프론트는 선행지표 이름 옆 아이콘 + 툴팁 인터랙션으로 최소 구현한다.
- 툴팁은 문제 케이스에만 노출하고, 잘하고 있는 지표는 침묵하는 방향으로 유지한다.
- 효과 검증이 필요해지면 후속으로 가이드 노출/클릭/기록 전환 이벤트를 설계한다.
