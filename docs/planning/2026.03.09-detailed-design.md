# DOWIN 상세 설계 및 기획

## 1~3. 배경/우선순위/기대효과

해당 내용은 `docs/planning/2026.03.09-service-overview.md`의 내용과 동일하므로 해당 문서를 참조합니다.

## 4. 상세 기능 설계 (Functional Specifications)

- **구조**: `User (사용자) > Workspace (워크스페이스/모임) > Scoreboard (점수판)`
- **워크스페이스**:
  - 현재는 **1인당 1개의 워크스페이스**만 생성/소속 가능하도록 제약 (MVP).
  - 추후 여러 모임에 참여할 수 있도록 다중 워크스페이스 확장성 고려.

### 4.1. 점수판 (Scoreboard Concept)

- **구성**: `목표 (Goal) > 후행지표 (Lag Measure) > 선행지표 (Lead Measure)`가 하나의 점수판(Scoreboard)으로 정의됨.
- **생명주기 (Lifecycle)**:
- **진행 중 (Active)**: 현재 추적 중인 점수판. **사용자당 하나의 워크스페이스에 단 하나의 활성 점수판만 존재 가능.** 선행지표의 추가/삭제가 자유로움.
  - **[Post-MVP] 종료/보관 (Archived)**: 목표 달성 혹은 목표 변경 시 점수판 전체를 종료 처리. 종료된 점수판의 데이터는 수정이 불가능하며 '과거 기록'으로 보관됨.
  - **새로운 시작**: 활성 점수판을 종료한 후에만 새로운 점수판을 생성할 수 있음.

### 4.1.1. 대시보드 (Scoreboard Interface)

- **나의 주간 점수판 (My Table)**:
  - 제공된 샘플 이미지와 같은 `선행지표 | 목표 | 월 | 화 | 수 | 목 | 금 | 토 | 일 | 달성도` 테이블.
  - 클릭 한 번으로 O/X 전환 (Notion 스타일의 간결한 인터랙션).
  - **팀 공유 범위**: 팀원이 서로를 볼 때는 **'이번 주(월~일)'**의 선행지표와 달성 현황만 노출하여 현재의 집중도를 극대화함.
  - **현재 활성화된 점수판**의 선행지표들만 노출됨.

### 4.1.2. 개인 대시보드 위젯 (Personal Dashboard Widgets)

- **위젯 기반 구성**: 나의 세부 지표와 통계를 위젯 형태로 자유롭게 대시보드에 배치.
- **주요 위젯 항목**:
  - **비교 분석 위젯**: 지난주 대비/지난달 대비 달성률 변화 시각화.
  - **기간별 요약 위젯**: 이번 주/이번 달 누적 달성도 및 진행률.
  - **동적 차트 위젯**: 위젯 내에서 차트 유형(선, 막대, 파이 등)을 동적으로 변경하여 데이터 확인.

### 4.1.2. 입력 보조 및 검증 (Input Assistance)

- **선행지표 가이드 (Lead Measure Guide)**: 지표 입력 시 '수치화 가능한가?', '직접 통제 가능한가?' 등의 4DX 원칙에 기반한 입력을 돕는 가이드 텍스트/툴팁 제공.
- **[Post-MVP] AI 실행계획 리뷰**: 점수판 생성/수정 시 AI가 DOWIN, 후행지표, 선행지표의 4DX 적합성(정렬, 통제 가능성, 측정 가능성)을 리뷰하고 개선 방향 제안.
  - 예: "목표는 분명하지만 선행지표가 DOWIN와 직접 연결되는 방식이 문장에서 잘 드러나지 않습니다."
  - 예: "선행지표가 행동보다 결과에 가까워 실제 실행 점검이 어렵습니다."

### 4.2. 지표 상세 (Measure Specifications)

- **선행지표 (Lead Measure)**:
  - 활성 번들 내에서 자유롭게 추가/삭제 가능.
  - 삭제 시 해당 번들 내의 과거 요일별 기록도 함께 삭제됨 (번들 자체가 히스토리이므로, 번들 내에서의 삭제는 물리적 삭제로 간주).
  - 지표별 `status` (Active/Archived) 필드를 통해 현재 '활성' 상태인 지표만 이번 주 승패에 반영함.

### 4.3. 사용자 및 인증 (User & Authentication)

- **가입 방식**: 일반적인 회원가입(SignUp) 기능은 제공하지 않음. 관리자(또는 소모임장)가 팀원들의 ID를 직접 생성하여 배포함.
- **초기 접속 및 프로필 설정**:
  - 초기 비밀번호는 전원 `user`로 설정.
  - **비밀번호 변경**: 강제 프로세스는 아니며, 최초 로그인 시 비밀번호 변경을 권장하는 토스트 알림 노출. 메뉴/마이페이지에서 상시 변경 가능.
  - **닉네임(Nickname) 설정**: 팀 내 식별을 위해 초기 닉네임 설정을 유도하거나 마이페이지에서 편집 가능.
  - **Preset Avatar 선택 (MVP 확장안)**:
    - 사용자 업로드 이미지 대신 서비스가 제공하는 preset avatar 중 하나를 선택한다.
    - avatar는 외부 storage가 아닌 정적 asset 또는 코드 기반 SVG로 제공한다.
    - 서버/DB에는 asset URL이 아니라 `avatarKey`만 저장한다.
    - preset은 브랜드 가이드에 맞는 동일 규격의 icon family로 운영한다.
- **개인/팀 뷰 전환 (View Toggle)**:
  - **전체 공개 원칙 (MVP)**: 팀의 동기부여와 성취 공유가 목적이므로, 동일 워크스페이스 내 활성화된 모든 선행지표와 달성 현황은 팀원들에게 상시 공개됨.
  - **[Post-MVP] 공개 범위 설정**: 본인의 세부 로그 공개 여부(전체 공개 vs 요약 공개 vs 비공개)를 사용자가 직접 설정할 수 있는 기능 제공.
  - **Mobile-First Responsive**: 노션과 같이 모바일에서도 테이블 형태의 기록이 편리하도록 반응형 레이아웃(Responsive Layout) 설계. 세로형 점수판 뷰 제공.

### 4.4. 상호작용 (Interaction)

- **기록 리마인드**:
  - 웹 기반 서비스의 한계를 고려하여 **PWA(Progressive Web App)** 기술을 통한 가벼운 푸시 알림 검토.
  - 또는 초기 단계에서는 사용자가 접속했을 때 "어제 기록이 빠져있네요!" 같은 인앱 알림/토스트 메시지로 대체.
- **[Post-MVP] 데이터 내보내기**: 선택한 기간의 요약/선행지표별 집계/일자별 상세를 엑셀(Excel) 파일로 다운로드할 수 있는 기능 제공.

### 4.5. 사용자 여정 및 화면 흐름 (User Journey & Flow)

1. **인증 프로세스**: 로그인 서버 체크 -> 최초 로그인 시 비밀번호 변경/닉네임 설정 권장 토스트 노출 -> 메인 대시보드 진입.
2. **점수판 유무에 따른 메인 화면**:
   - **점수판 없음**: '새 점수판 만들기' 버튼이 강조된 화면 노출. 클릭 시 목적/지표 입력 폼으로 이동. (팀원들도 점수판이 없는 경우 각자의 영역에 동일하게 노출)
   - **점수판 있음**: 현재 활성화된 점수판의 주간 현황 노출.
3. **대시보드 상호작용**:
   - **뷰 토글**: `나의 지표` vs `팀 전체 지표` 스위칭. (팀 뷰는 전원 이번 주 데이터만 요약 노출)
   - **상세 이동**: 요약된 선행지표 클릭 시 해당 지표의 전체 히스토리 및 수정 화면으로 이동.
   - **번들 관리**: 현재 목표 달성 시 '축하와 함께 종료(Archive)' 혹은 '목표 수정' 옵션 제공.

### 4.6. 시각화 전략 (Visualization Strategy)

- **점수판 구성 표현**: 대시보드 최상단에 `[목표] -> [후행지표]`를 카드 형태로 고정 노출하여 서비스 접속 시마다 최종 목적지를 리마인드함.
- **점수판 (Scoreboard)**:
  - **MVP**: 간결하게 요일별 O/X 상태만 노출.
  - **팀원 카드**: 팀 전체 뷰에서는 팀원들의 프로필과 함께 '이번 주 승패 요약(Win/Loss)'을 카드 형태로 나열하여 한눈에 에너지 파악.

### 4.7. Preset Avatar 상세 설계 (2026-03-18)

- **목표**: 사용자의 식별성을 높이되, 사진 업로드/스토리지 운영 없이 가볍게 관리 가능한 프로필 이미지 체계를 만든다.
- **자산 관리 방식**:
  - 자산 위치는 `public/assets/avatars/*` 같은 정적 경로를 기본안으로 둔다.
  - 포맷은 SVG 우선으로 하여 다중 해상도 대응과 용량 최소화를 확보한다.
  - 프론트는 `avatarKey -> asset path` 또는 `avatarKey -> SVG component` 매핑 레이어를 가진다.
- **데이터 구조 원칙**:
  - `users.avatar_key` 추가
  - 기존 사용자 호환을 위해 초기에는 `NULL` 허용
  - `NULL`은 "preset 미선택" 상태를 의미하며 기존 fallback 아이콘으로 렌더링
  - 허용값 예시: `NULL`, `avatar-01` ... `avatar-12`
- **생성 규칙**:
  - 둥근 프로필 마스크 안에서도 실루엣이 유지되는 구조여야 한다.
  - 1px 내외 stroke, 얇은 라인, 제한된 면 분할을 기본 규칙으로 둔다.
  - Pure White 또는 Cold Off-white 배경에서도 묻히지 않도록 외곽 ring 또는 contrast layer를 포함한다.
  - Lucide 아이콘과 충돌하지 않도록 "툴 UI 아이콘"이 아니라 "사용자 토큰"에 가까운 스타일로 만든다.
- **선택 UX**:
  - 프로필 화면에서 선택 가능한 썸네일 grid 제공
  - 현재 선택된 avatar는 indigo border 또는 background 변화로 표시
  - 변경 즉시 미리보기를 보여주고, 저장 전후 상태 차이를 분명히 한다
- **확장 고려(Post-MVP)**:
  - preset 추가 배포
  - seed 기반 자동 조합
  - role/workspace badge overlay

### 4.8. 동기부여 요소 (Gamification & Motivation)

- **달성 축하 효과 (Celebration)**: 오늘의 지표 혹은 점수판 목표 달성 시 **Confetti(폭죽 애니메이션)** 효과를 노출하여 성취감 극대화.
