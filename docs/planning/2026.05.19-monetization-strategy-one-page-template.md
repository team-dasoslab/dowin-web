# 수익화 전략 1페이지 템플릿

## 1. 이 문서의 역할

- 이 문서는 가격표를 바로 정하는 문서가 아니다.
- 먼저 `누가`, `왜`, `무엇에`, `어떤 방식으로` 돈을 낼지를 짧게 정리하는 문서다.
- 제품 초기, 신규 사업 검토, 기능 우선순위 조정 전에 먼저 작성하는 것을 권장한다.
- 이 템플릿은 하나의 단일 정식 프레임워크를 그대로 옮긴 문서가 아니라, 실무에서 널리 쓰이는 가격/패키징 원칙을 1페이지 작성용으로 압축한 합성 템플릿이다.

## 2. 작성 원칙

- 기능 설명보다 `고객이 돈을 내는 이유`를 먼저 적는다.
- 사용자와 결제자가 다를 수 있으면 반드시 분리해서 적는다.
- 추상 표현 대신 시간, 비용, 리스크, 매출 같은 손실 언어로 적는다.
- 확신이 없는 항목은 단정하지 말고 `가설`로 표시한다.

## 4. 빠른 작성용 체크리스트

- 고객이 충분히 좁혀져 있는가
- 세그먼트별 지불 의사 차이가 드러나는가
- 문제의 방치 비용이 숫자나 손실 언어로 적혀 있는가
- 사용자와 결제자를 구분했는가
- 현재 대체재가 적혀 있는가
- 과금 단위가 고객 가치와 연결되는가
- 가치 지표가 이해 가능하고, 예측 가능하고, 통제 가능한가
- 무료와 유료의 경계가 기능 나열이 아니라 문제 크기 차이로 설명되는가
- 패키징이 업셀 경로를 만들고 있는가
- 검증 계획이 가격 토론이 아니라 실제 구매 맥락 검증으로 이어지는가

## 5. 자주 하는 실수

- 가격부터 정한다
- 고객을 너무 넓게 잡는다
- 세그먼트별 지불 의사 차이를 무시한다
- 기능 개수를 가치로 착각한다
- 현재 대체재를 무시한다
- 사용자와 결제자가 같다고 가정한다
- value metric을 내부 운영 단위로 잡는다
- 업셀 이유 없이 티어만 늘린다
- 설문만 하고 실제 결제 맥락을 보지 않는다

## 7. 이 템플릿을 쓴 뒤 다음 단계

1. 인터뷰 대상 5명 이상을 잡는다.
2. 실제 대체재와 지출 패턴을 확인한다.
3. 가격이 아니라 구매 이유부터 반복해서 검증한다.
4. 그 다음에야 플랜 구조와 가격표 초안을 만든다.

## 8. 근거가 되는 대표 방법론과 참고 링크

### 8.1. 이 템플릿이 기대는 공통 실무 흐름

- `고객 가치 정의 -> value metric 선택 -> pricing model 선택 -> tier/package 설계 -> 성과 측정` 순서는 대표적인 SaaS pricing 실무 흐름과 맞춘 것이다.
- `사용자와 결제자/승인자 분리`는 특히 B2B pricing/packaging 설계에서 자주 쓰이는 관점이다.
- `가격 조사`는 단일 방식으로 끝내지 않고, 인터뷰와 가격 조사 기법을 함께 쓰는 것이 일반적이다.

### 8.2. 섹션별 참고 출처

- `3.1 한 줄 정의`, `3.4 해결하는 문제`, `3.6 돈을 내는 이유`
  - 가치 기반 가격(Value-based / Value-driven pricing) 관점 참고
  - Stripe: [Value-driven pricing strategy](https://stripe.com/us/resources/more/value-driven-pricing)

- `3.3 결제자`
  - 사용자, 리더, 예산 승인자 분리 관점 참고
  - AcuStrategy: [SaaS Pricing And Packaging: A Simple Guide](https://acustrategy.com/blog/saas-pricing-and-packaging/)

- `3.7 가치 지표`, `3.8 가격 모델 가설`, `3.9 패키징 가설`, `3.10 업그레이드 트리거`
  - value metric, pricing model, tier structure, upgrade path 관점 참고
  - Stripe: [SaaS pricing and packaging strategy](https://stripe.com/ae/resources/more/saas-pricing-and-packaging-strategy)

- `3.11 검증 계획`
  - 인터뷰 기반 검증은 위 Stripe/AcuStrategy 흐름을 참고했고, 가격 민감도 조사 자체는 아래 기법을 참조했다.
  - Van Westendorp 개요: [Price Sensitivity Meter PDF](https://www.gaits.org/documents/20151/228860/Van%20Westendorp%E2%80%99s%20Price%20Sensitivity%20Meter.pdf/c6efd7d4-56e2-dacb-1eae-ad9c8c5e572d)
  - Gabor-Granger 개요: [Umbrex Gabor-Granger Method](https://umbrex.com/resources/frameworks/pricing-frameworks/gabor-granger-method/)

### 8.3. 해석 주의

- 이 문서는 특정 회사의 정식 프레임워크를 그대로 복제한 문서가 아니다.
- 더 정확히는 아래를 합쳐서 실무용 템플릿으로 재구성한 것이다.
  - 가치 기반 가격 사고방식
  - value metric 설계
  - pricing/package/tier 설계
  - 인터뷰 및 가격 민감도 조사 기법
- 따라서 이 문서는 `정식 이론 문서`라기보다 `초기 전략 정리용 working template`로 쓰는 편이 맞다.
