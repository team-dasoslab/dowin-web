# 프로모션 코드 기한 제한 기능 기획

## 1. 배경 및 문제 정의
현재 마케팅 초대코드는 무제한 기간 동안 사용할 수 있으며, 제공되는 혜택 역시 "Basic 영구 무료 혜택(Lifetime Basic)"으로 고정되어 있습니다. 
초기 피드백 수집 목적의 영구 혜택에서 나아가, 유효 기간이 있는 코드(희소성 부여) 및 기간 한정 무료 체험(Trial) 용도로 프로모션 코드를 확장하기 위해 기한 제한 기능이 필요합니다.

## 2. 기한 제한의 2가지 관점 (모두 포함)
"기한 제한"은 시스템 상 다음 두 가지로 나뉘며, 본 기획에서는 **두 가지 모두**를 지원하는 구조로 구현합니다.

1. **코드 사용 만료 기한 (Code Expiration)**
   - 예: "이 코드는 발급일로부터 1주일 내에만 사용할 수 있습니다."
   - 목적: 프로모션의 희소성 및 긴급성 조성.
2. **혜택 유지 기한 (Entitlement Duration)**
   - 예: "이 코드로 가입하면 1주일 동안만 Basic 혜택을 무료로 사용할 수 있습니다."
   - 목적: 영구 무료가 아닌 '기간 한정 체험판(Free Trial)' 제공.

## 3. 스키마 변경 계획
`marketing_invite_codes` 테이블에 다음 필드를 추가합니다.

- `expires_at` (timestamp, nullable): 코드 자체가 만료되어 더 이상 사용할 수 없는 시각.
- `entitlement_duration_days` (integer, nullable): 코드를 사용한 시점부터 혜택이 유지되는 일수. 값이 없으면 기존처럼 영구 혜택.

*참고: 혜택 유지 기한이 설정된 코드를 사용했을 때, 워크스페이스의 과금 만료일(billing state)에 반영되어 일정 기간 이후 권한이 회수되거나 유료 전환을 요구할 수 있게 합니다.*

## 4. 어드민 콘솔 정책
어드민 콘솔의 프로모션 코드 생성 화면에서 다음 두 가지 옵션을 각각 설정할 수 있게 합니다.

- **코드 만료 기한 (선택)**
  - 제한 없음, 1주, 2주, 1개월, 2개월 등
- **혜택 유지 기한 (선택)**
  - 영구 혜택 (제한 없음), 1주, 2주, 1개월, 2개월 등

목록 및 상세 화면에서도 해당 기한 및 만료 여부 상태를 명확히 표시합니다.

## 5. 예상 사용자 흐름
1. **코드 사용 시점 (만료 체크)**
   - 사용자가 코드를 입력할 때 `expires_at`이 지났다면 "만료된 프로모션 코드입니다" 에러를 노출합니다.
2. **혜택 부여 시점**
   - 혜택 유지 기한이 설정된 코드인 경우, 생성된 워크스페이스의 과금 상태에 남은 기간이 노출됩니다.
   - 예: `Basic 혜택 (O월 O일까지 무료)`

## 6. MVP 범위
- DB 스키마 업데이트 (`expires_at`, `entitlement_duration_days` 추가)
- API 업데이트 (어드민 생성, 상세, 사용자 사용 검증)
- 어드민 생성 UI에 만료 기한 및 혜택 유지 기한 선택 폼 추가
- 어드민 상세 UI에서 기한 상태 표시
- 사용자 코드 입력 시 기한 만료 에러 메시지 처리 (en/ko 메시지 추가)
