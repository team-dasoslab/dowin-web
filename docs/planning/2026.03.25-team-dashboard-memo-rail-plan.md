# 팀 대시보드 메모 레일 확장안

## 1. 배경 및 맥락 (Background & Context)

- 현재 팀 대시보드는 이번 주 점수판 현황을 읽기 전용으로만 보여주기 때문에, 회의 중 나온 코멘트나 다음 액션을 사람별로 바로 붙잡아두기 어렵다.
- 실제 팀 대시보드 프로토타입을 다듬는 과정에서, 주간 점수판 옆에서 사람별 메모를 남기고 다시 열어보는 흐름이 회의 맥락에 가장 잘 맞는다는 점이 확인됐다.
- 별도 노트 앱이나 채팅으로 빠지지 않고, 점수판 맥락 안에서 짧은 피드백을 남길 수 있어야 한다.

## 2. 영향 도메인 (Domain Impact)

- `dashboard` 프론트엔드 인터랙션
- 향후 `dashboard` 또는 `comment/memo` 성격의 저장 계약
- profile avatar 재사용 규칙

## 3. MVP 범위 (MVP Scope)

- 팀 주간 점수판에서 사용자별 `메모하기` 버튼을 제공한다.
- `메모하기`를 누르면 해당 사용자 블록 기준으로 오른쪽 메모 레일이 열리고, 입력창이 노출된다.
- 이미 메모가 존재하는 사용자는 `메모보기` 버튼으로 입력창 없이 해당 사용자 메모만 다시 열어볼 수 있어야 한다.
- 한 번에 여러 사람 메모를 동시에 펼치지 않고, 현재 선택된 한 사용자 레일만 표시한다.
- 메모 카드에는 작성자 avatar, 닉네임, 상대 시간, 완료 체크, 삭제 액션을 노출한다.
- 완료 체크는 `해결됨` 또는 `확인됨` 상태를 표현하는 UI 액션으로 처리한다.

## 4. MVP 제외 범위 (Non-Goals)

- 실시간 동시 편집
- 멘션, 파일 첨부, 반응, 좋아요
- 쓰레드형 대댓글
- 서버 저장 전 권한 정책 확정 전의 팀 전체 영구 보관
- 메모 검색, 필터, 정렬 옵션

## 5. 핵심 UX 원칙 (UX Principles)

- 메모는 점수판을 가리지 않는 오른쪽 보조 레일로 보여야 한다.
- 메모 레일이 열릴 때는 대시보드 본문이 약간 왼쪽으로 이동해 공간을 만든다.
- `메모하기` 버튼 자체는 사용자가 누른 위치를 잃지 않도록 최대한 제자리 감각을 유지해야 한다.
- 메모 레일은 현재 선택된 사용자 기준으로만 열리므로, 어떤 메모를 보고 있는지 맥락이 분명해야 한다.
- 메모 입력창은 과한 협업 도구처럼 보이지 않도록 최소 입력과 등록 액션 중심으로 단순하게 유지한다.
- avatar는 팀 대시보드와 프로필의 `avatarKey` 규칙을 재사용한다.

## 6. 상태 모델 원칙 (State Model)

- `메모하기`와 `메모보기`는 같은 레일을 공유하되, 모드만 다르게 본다.
- `메모하기`는 입력창과 기존 메모 목록을 함께 보여준다.
- `메모보기`는 입력창 없이 메모 목록만 보여준다.
- 다른 사용자 버튼을 누르면 이전 사용자 레일은 닫히고 새 사용자 레일로 전환된다.
- 메모가 하나도 없는 사용자는 `메모보기`를 노출하지 않는다.

## 7. 레이아웃 원칙 (Layout Principles)

- 기본 팀 대시보드 본문 폭은 유지한다.
- 메모 레일은 본문 내부 컬럼이 아니라, 본문 오른쪽 바깥에 떠 있는 보조 레일처럼 보여야 한다.
- 레일 높이와 간격 충돌은 구조적으로 관리해야 하며, 단순 `padding-bottom` 보정보다 레이아웃 의미가 유지되는 방식이 우선이다.
- 모바일에서는 오른쪽 레일 고정보다 세로 스택 전환이 현실적이므로, 좁은 화면에서는 메모 영역이 점수판 아래로 자연스럽게 내려와도 된다.

## 8. 저장 전략 원칙 (Persistence Principles)

- 프로토타입 단계에서는 로컬 UI 상태로 상호작용을 검증해도 된다.
- 서버 저장을 붙일 때는 사용자별 팀 메모의 소유권, 공개 범위, 수정/삭제 권한을 먼저 정의해야 한다.
- 저장 계약이 생기면 `dashboard` 집계 응답에 직접 섞기보다 메모 전용 조회/변경 계약으로 분리하는 것이 안전하다.

## 9. 성공 및 실패 기준 (Success & Failure Criteria)

### 성공

- 팀 회의 중 특정 사용자 점수판을 보다가 바로 메모를 남길 수 있다.
- 어떤 사용자의 메모를 보고 있는지 혼동되지 않는다.
- 메모 레일을 열어도 팀 대시보드 본문 가독성이 무너지지 않는다.
- 버튼, 레일, 카드 액션이 과도한 협업 툴처럼 보이지 않고 DOWIN의 현재 제품 톤을 유지한다.

### 실패

- 여러 사용자 메모가 한 화면에서 겹치거나 컨텍스트가 섞인다.
- 메모 레일 때문에 가로 스크롤이 생기거나 본문 정보가 과도하게 밀려난다.
- `메모하기`와 `메모보기`의 차이가 사용자가 이해하기 어렵다.
- 메모 카드 액션이 너무 많아져 핵심 점수판보다 시선을 빼앗는다.
