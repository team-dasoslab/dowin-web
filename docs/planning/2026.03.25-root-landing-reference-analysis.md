# 루트 랜딩 레퍼런스 분석

## 1. 대상 레퍼런스

- 사용자가 첨부한 세로형 랜딩 참고 이미지
- 전반 인상:
  - 밝은 배경 위 대형 타이포
  - 블러/그라데이션 필드
  - 비대칭 섹션 구성
  - 텍스트보다 레이아웃 리듬이 먼저 읽히는 구조

## 2. 이 이미지가 좋아 보이는 이유

### 2.1. 첫 화면에서 시선이 바로 고정된다

- 큰 헤드라인이 가장 먼저 보인다.
- 그 다음에 오른쪽 비주얼, 마지막으로 CTA가 읽힌다.
- 즉, 사용자가 `무엇인지 이해 -> 감각적으로 납득 -> 행동` 순서로 이동한다.

### 2.2. 섹션마다 강조점이 하나뿐이다

- 어떤 섹션은 큰 이미지
- 어떤 섹션은 큰 숫자
- 어떤 섹션은 좌우 비대칭 카드
- 어떤 섹션은 로고 월
- 모든 구간이 “설명”이 아니라 “주인공 하나”를 가진다.

### 2.3. 텍스트가 많지 않은데도 풍부하게 느껴진다

- 문장이 길지 않다.
- 대신 여백, 크기 차이, 이미지 블록, 배경 색면이 리듬을 만든다.
- 내용량이 아니라 `배치 감각`으로 페이지를 풍성하게 보이게 한다.

### 2.4. 같은 레이아웃을 반복하지 않는다

- Hero: 좌 텍스트 / 우 비주얼
- 다음 섹션: 넓은 배경 필드
- 다음 섹션: 좌 리스트 / 우 이미지
- 다음 섹션: 숫자 강조
- 다음 섹션: 카드 그리드
- 반복성이 아니라, 섹션 전환감이 페이지 체류를 만든다.

## 3. 화면 구조 분석

### 3.1. Hero

- 특징:
  - 큰 타이포 2~3줄
  - 짧은 설명
  - CTA 1개
  - 오른쪽 혹은 뒤쪽에 큰 비주얼
- 핵심:
  - 첫 화면은 서비스 소개보다 `인상 형성`이 우선이다.
  - 텍스트 양은 적고, 비주얼 비중은 크다.

### 3.2. 전환용 비주얼 구간

- Hero 바로 아래에 색면/블러 구간이 들어간다.
- 이 구간은 정보를 설명하기보다 페이지 흐름을 환기한다.
- 사용자는 이때 “브랜드 톤”을 감각적으로 받아들인다.

### 3.3. 비대칭 정보 구간

- 좌측은 짧은 라벨/카테고리
- 우측은 이미지나 카드
- 혹은 반대로 큰 숫자/강조 정보가 한쪽에 몰린다.
- 핵심은 모든 섹션이 동일한 그리드로 보이지 않는다는 점이다.

### 3.4. 신뢰/확장 구간

- 최근 프로젝트, 클라이언트 로고, 사례 등
- 콘텐츠 양이 많아도 “리스트 설명”보다 “배열된 시각 요소”로 보인다.

## 4. 시각 메커니즘 분석

### 4.1. 큰 타이포 대비

- 헤드라인은 화면에서 과감하게 크게 쓴다.
- 본문은 매우 짧고 작게 유지한다.
- 큰 제목과 작은 설명의 대비가 페이지를 세련되게 만든다.

### 4.2. 블러/그라데이션은 배경이지 주연이 아니다

- 색 필드는 장식용 배경이다.
- 카드나 텍스트보다 앞에 나오지 않는다.
- 즉, 색 자체가 아니라 `깊이와 공기감`을 만드는 장치다.

### 4.3. 카드와 섹션의 위계가 명확하다

- 어떤 카드는 정보
- 어떤 카드는 장식
- 어떤 카드는 숫자 강조
- 모든 요소가 같은 중요도로 보이지 않는다.

### 4.4. 여백이 콘텐츠를 설명한다

- 섹션 사이 간격이 크다.
- 그래서 사용자는 “새 섹션”을 눈으로 먼저 감지한다.
- 텍스트가 적어도 허전해 보이지 않는 이유는 이 여백 때문이다.

## 5. DOWIN에 가져와야 할 요소

### 5.1. 가져올 것

- Hero에서 큰 타이포와 큰 제품 비주얼의 강한 대비
- 정보 섹션 사이에 쉬어가는 색 필드/블러 필드
- 섹션마다 대표 포인트 하나만 두는 구성
- 비대칭 레이아웃
- 큰 숫자, 큰 키워드, 큰 카드 같은 단일 강조 장치
- 페이지 전체에 공기감이 느껴지는 넓은 여백

### 5.2. DOWIN용으로 번안할 것

- 레퍼런스의 비주얼 박스/이미지 카드 대신 Dowin 대시보드 조각 사용
- 클라이언트/프로젝트 섹션 대신 DOWIN의 실행 흐름 블록 사용
- 서비스 소개용 추상 키워드 대신 DOWIN의 제품 구조 키워드 사용
  - `Dowin`
  - `Lead`
  - `Score`
  - `Review`

## 6. DOWIN에 그대로 가져오면 안 되는 요소

### 6.1. 과한 다색 사용

- Dowin 브랜드는 `Linear Light Aesthetic` 기반이다.
- 레퍼런스처럼 다채로운 색을 전면에 쓰면 DOWIN의 정교함이 무너질 수 있다.
- DOWIN는 인디고 중심, 저채도 보조색만 제한적으로 써야 한다.

### 6.2. 장식용 이미지 남발

- 레퍼런스는 에이전시/디자인 포트폴리오 톤이라 이미지 블록이 많다.
- DOWIN는 제품 랜딩이므로 추상 이미지보다 실제 제품 조각이 더 중요하다.

### 6.3. 브랜딩보다 연출이 앞서는 구성

- DOWIN는 “예쁜 사이트”보다 “정교한 프로 도구”로 보여야 한다.
- 따라서 모든 화려한 연출은 제품 이해를 돕는 수준까지만 허용된다.

## 7. Dowin 랜딩에 적용할 구체 원칙

### 7.1. Hero 규칙

- 좌측:
  - 큰 헤드라인 2~3줄
  - 본문 1문장
  - CTA 1~2개
- 우측:
  - 큰 제품 프리뷰 1개
  - 카드/배지/블러는 보조 요소

### 7.2. 섹션 규칙

- 한 섹션당 한 메시지만 남긴다.
- 설명 문단은 최대 2문장
- 각 섹션의 시각적 주인공은 아래 중 하나만 선택한다.
  - 큰 카드
  - 큰 숫자
  - 큰 키워드
  - 큰 제품 조각

### 7.3. 컬러 규칙

- 기본:
  - White
  - Sub-background
  - Indigo
- 보조:
  - 저채도 mint
  - 저채도 peach
  - 매우 약한 lavender tint
- 금지:
  - 과포화 네온
  - 무거운 다크 배경
  - 다색 gradient를 전면 주연으로 사용하는 것

### 7.4. 형태 규칙

- 기본 radius는 브랜드 문서 기준으로 작게 유지하되, 랜딩의 대표 쇼케이스 블록만 약간 크게 허용한다.
- 대부분의 구분은 1px border로 처리한다.
- 섀도우는 얕고 넓게만 사용한다.

## 8. 추천 랜딩 구조안

### 8.1. 섹션 순서

1. Hero
   - 큰 헤드라인
   - 짧은 설명
   - CTA
   - 큰 제품 비주얼
2. Visual Break
   - 블러/그라데이션 필드
   - 정보 없는 전환용 구간 또는 짧은 키워드 한 줄
3. Execution Loop
   - `01 Focus`
   - `02 Lead`
   - `03 Review`
4. Product Proof
   - 점수판 / 주간율 / 월간율 / 워크스페이스 흐름 조각
5. Footer CTA
   - 로그인

### 8.2. 제거 우선순위

- 긴 설명 문단
- 작은 보조 카드 다수
- 섹션마다 반복되는 설명성 텍스트
- 같은 형태의 3열 카드 반복

## 9. 성공 판단 기준

- 첫 화면만 보고도 “실행 관리 제품”이라는 인상이 바로 생긴다.
- 텍스트를 다 읽지 않아도 페이지 리듬이 좋다.
- 섹션마다 무엇을 강조하는지 1초 안에 식별된다.
- 예쁘기만 한 랜딩이 아니라, DOWIN답게 정교한 프로 도구처럼 보인다.
