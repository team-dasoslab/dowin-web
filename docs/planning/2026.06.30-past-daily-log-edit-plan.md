# 지난주 Daily Log 수정 허용 기능 구현 계획

## Background
- 현재 Dowin은 당주(This Week)의 Daily Log만 수정/입력할 수 있도록 서버에서 엄격하게 제한(`assertPastWeekLogEditable`)하고 있음.
- 사용자가 이전 주나 그 과거의 기록을 늦게 입력해야 할 때 제약이 발생함.
- 관리자 권한으로 설정에서 이를 해제할 수 있는 유연성을 제공하기 위함.

## Scope (MVP)
- `workspaces` 테이블에 `allowPastDailyLogEdit` 플래그 추가 (기본값 false).
- 워크스페이스 설정 페이지에서 해당 플래그를 관리자만 On/Off 할 수 있는 토글 제공.
- Daily Log 저장, 삭제 로직에서 해당 플래그를 참조하여 과거 시간 제한 우회.
- 프론트엔드 Daily Log Grid의 비활성화(disabled) 조건에 이 플래그 반영.

## Action Items
1. **DB Schema & Migration**: `allowPastDailyLogEdit` 추가
2. **API Layer**: `/api/workspaces/[workspaceId]` PUT 라우트 및 OpenAPI `workspaceUpdateSchema` 변경
3. **Service Layer**: `daily-log.service.ts`에서 `assertPastWeekLogEditable` 시 워크스페이스 설정 반영
4. **UI**: 워크스페이스 Settings 화면에 토글 스위치 추가, Daily Log 화면에서 UI 반영

## Cost and Estimation
- 스키마 변경, API 업데이트, Service 수정, UI 반영 등 단일 사이클 내 해결 가능.

## Success Criteria
- 워크스페이스 관리자가 설정 페이지에서 "과거 기록 수정 허용"을 켤 수 있음.
- 이 설정이 켜지면 지난주/과거 Daily Log 입력 및 삭제 시 403 에러가 발생하지 않음.
- 일반 멤버나 설정이 꺼진 상태에서는 여전히 과거 기록이 블로킹 됨.
