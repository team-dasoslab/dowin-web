# 팀원 간 소셜 넛지(콕 찌르기) 도입 기획안

## 1. Background & Context (도입 배경)
Dowin은 팀 주간 성과 관리와 개인이 가장 중요한 목표(Do What's Important Now)에 집중하도록 돕는 서비스입니다. 현재 주간 달성률 및 연속 달성 스트릭, 그리고 최근 도입된 일일 접속 스트릭을 통해 개인의 동기를 부여하고 있습니다.
하지만 팀원들이 서로의 진행 상황을 투명하게 볼 수 있음에도 불구하고, 목표 달성을 잊거나 미루고 있는 팀원에게 액션을 취할 수 있는 수단이 부족합니다. 카카오톡이나 슬랙을 통한 리마인드는 자칫 딱딱한 업무 지시처럼 느껴질 수 있습니다. 
따라서 대시보드 내에서 가볍고 긍정적인 압박(Peer Pressure)을 형성할 수 있는 **'콕 찌르기(Nudge)'** 기능을 도입하고자 합니다.

## 2. Priority & Expected Impact (기대 효과)
- **리텐션 및 활동률 증가:** 찌르기를 받은 유저는 푸시 알림을 통해 즉각적으로 서비스로 진입하게 됩니다.
- **긍정적 팀 문화 형성:** "너 왜 안 해?"라는 텍스트 대신 `[Dowin] {sender}님이 콕 찔렀어요!` 라는 가벼운 알림을 통해, 서로 챙겨주는 분위기를 만듭니다.
- **낮은 도입 비용:** 기존 대시보드 MemberCard와 연동되어 빠르게 도입 가능하며, 즉각적인 반응을 이끌어 낼 수 있습니다.

## 3. Scope & MVP

### MVP (이번 작업 범위)
- **버튼 동작 및 UI:** 
  - `MemberCard` 우측 상단에 텍스트 버튼(`콕 찌르기`) 형태로 노출.
  - 상대방의 지표 입력 여부와 상관없이 심플하게 '👉 콕 찌르기' 단일 액션으로 통일.
  - 실수(오터치)를 방지하기 위해 클릭 시 브라우저 기본 `confirm`("ㅇㅇㅇ님을 콕 찌르시겠습니까? 상대방에게 푸시 알림이 전송됩니다.") 호출.
- **도배 방지 (Rate Limit):** 
  - (제거됨) 횟수 제한 없이 자유롭게 찌르기를 허용합니다. 불필요한 이력 저장(DB) 비용을 아끼고, 찌르기 액션의 가벼움을 유지합니다.
- **푸시 알림 전송 로직:**
  - `fcm.ts` 서비스를 활용해 타겟 유저에게 푸시 알림 전송.
  - 푸시 문구: `[Dowin] {sender}님이 콕 찔렀어요!`
- **DB 스키마:**
  - 횟수 제한을 없앰에 따라 별도의 찌르기 기록 테이블(`member_nudges`)은 생성하지 않습니다.

### Post-MVP (향후 확장 범위)
- **찌름 응답(Reaction):** 찌르기를 받은 사람이 앱 내에서 즉각적으로 "지금 하러 갈게요 🏃" 같은 퀵 리액션을 보낼 수 있는 기능.
- **찌르기 랭킹:** 누가 팀을 가장 많이 챙겼는지 보여주는 재미 요소 랭킹.

## 4. Action Items (다음 단계)
- [ ] **[Backend] API 설계 및 구현**: `POST /api/workspaces/{workspaceId}/members/{userId}/nudge` API 스펙 추가 (OpenAPI 갱신) 및 푸시 서비스 연동.
- [ ] **[Backend] FCM 연동 로직 검증**: 실제 찌르기 API 호출 시 FCM 전송 함수 호출 여부 테스트 작성.
- [ ] **[Frontend] 클라이언트 생성**: `yarn gen:api`를 통한 API 훅 재생성.
- [ ] **[Frontend] API 연동**: 프로토타입으로 작성된 `MemberCard`의 `confirm` 승인 이후 실제 `useMutation`으로 Nudge API 쏘도록 로직 연결 및 성공 시 Toast 알림, 해당 유저 버튼 비활성화 상태 업데이트.

## 5. 결론 및 참고 사항
- 본 기획안은 그릴링(Grilling)을 통해 도배 방지 규칙, 버튼 UI 형태(Icon -> Text), Confirm 도입 등이 사전에 협의 및 확정되었습니다.
- 설계는 Dowin의 기존 룰(`AGENTS.md`)에 따라 무거운 시스템 대신 직관적이고 빠르게 적용할 수 있는 형태로 구성했습니다.
