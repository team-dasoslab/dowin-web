# 일일 체크인 스트릭(Daily Check-in Streak) 추가 기획

## 1. Background & Context (도입 배경)
앞서 구현된 **연속 달성 스트릭(Weekly Win Streak)**은 매주 목표의 80% 이상을 달성해야만 유지되는 강력한 게이미피케이션 요소입니다. 하지만 목표 달성 자체가 매우 어려운 경우, 초보 유저나 슬럼프에 빠진 유저는 80% 달성을 놓치는 순간 스트릭이 깨지고 이로 인해 오히려 서비스 이탈이나 동기부여 저하를 겪을 수 있습니다.

따라서 성과(달성률)와 무관하게, **"앱에 접속해서 지표를 점검하고 관리하는 행위"** 자체를 칭찬하고 보상하는 **일일 체크인 스트릭(Action/Login Streak)**이 추가로 필요합니다. 

## 2. Priority & Expected Impact (기대 효과)
- **높은 달성 빈도**: 매일 들어오기만 해도 숫자가 오르기 때문에, 강력한 리텐션 훅(Retention Hook)이 됩니다.
- **상호 보완적 시스템**: 
  - (A) 퀄리티 보상: "연속 n주 80% 달성!" (어려움, 명예로움) - 🔥 아이콘
  - (B) 꾸준함 보상: "연속 n일 접속 및 관리!" (쉬움, 일관성) - ⚡️ 아이콘
- **푸시 알림 최적화**: 단순히 "오늘 지표를 입력하세요"보다 "오늘 지표를 체크하지 않으면 N일째 이어온 스트릭이 0으로 초기화됩니다!"라는 손실 회피(Loss Aversion) 심리를 자극하는 푸시 알림 전송이 가능해집니다.

## 3. Scope & MVP vs Post-MVP

### MVP (이번 작업 범위) - **완료**
- **Daily Check-in Streak 로직**: 유저의 최근 활동 기록(대시보드 진입 기준)을 기준으로 며칠 연속으로 활동했는지 추적/저장합니다.
  - **구현 방식 결정**: 단순 정수 카운트 대신 `checkinStreakStartDate`와 `checkinStreakEndDate`를 `workspace_members`에 저장합니다.
  - 대시보드 조회 시 다음과 같이 날짜를 갱신합니다:
    1. 어제 접속 이력이 있으면 (`오늘 - 1일 == endDate`): `endDate = 오늘`로 갱신 (스트릭 유지)
    2. 이틀 이상 접속이 끊겼다면 (`오늘 - 2일 >= endDate`): `startDate = 오늘`, `endDate = 오늘`로 초기화 (스트릭 리셋)
- **대시보드 UI 노출**: 
  - 기존 주간 달성 불꽃 아이콘(🔥) 옆에 "⚡️ n일 연속 체크인" 형태로 노출.
  - 팀 대시보드의 멤버 카드에도 작게 (⚡️) 스트릭 뱃지 추가.

### Post-MVP (향후 확장 범위)
- **푸시 알림 통합**: 매일 지정된 시각에 일일 리마인더 발송 시, 스트릭 정보 포함. ("오늘 접속하지 않으면 N일 스트릭이 깨져요!")
- **점수판 성실도 랭킹**: 멤버별 달성률 랭킹뿐 아니라 성실도(스트릭) 노출 기능 확장.

## 4. Action Items (다음 단계)
1. **[Backend] 스트릭 인정 기준 및 계산/저장 방식 확정** - **[완료]**
   - 대시보드 조회 API(`getMyDashboard`, `getTeamDashboard`) 요청 시점을 "체크인"으로 간주하고 startDate, endDate 갱신 완료.
2. **[Frontend] UI 반영 위치 결정** - **[완료]**
   - My Dashboard 상단 및 Member Card 컴포넌트에 번개(⚡️) 아이콘으로 노출 완료.
3. **[Notification] 알림 문구 작성** - [대기중]
   - 스트릭이 깨지기 전 발송할 푸시 알림의 copy 작성 및 발송 로직 업데이트 (Post-MVP로 이관).

## 5. 결론 및 구현 히스토리
- **스트릭 인정 기준**: 단순 지표 입력(Daily Log)이 없는 요일도 있을 수 있으므로, **앱 접속(대시보드 로드)만으로도 인정**하도록 타협점을 찾았습니다.
- **날짜 계산 설계**: `currentCheckinStreak`를 직접 DB에 저장하는 대신 `startDate`와 `endDate`를 저장함으로써 날짜 기반의 동적 계산을 통해 무결성을 확보했습니다.
