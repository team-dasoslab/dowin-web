# 고객 발굴 세션 — 초기 페르소나 정의

## 배경

한 달간 수익화 시도가 지지부진한 상황에서, LLM 기반 리서치보다 실제 주변인 인터뷰를 통해 고객을 좁히기로 결정했다.

**Dowin의 핵심 목표 (재확인)**

- 팀이 명확한 목표를 설정하고 인지해 길을 잃지 않게 한다
- 경쟁이 아닌 서로의 상황을 파악하고 팀워크를 강화한다
- 리더는 팀원들이 액션 아이템을 잘 수행하고 있는지 파악한다

---

## 고객 유형 매트릭스 (2×2)

> X축: 목표 명확도 (낮음 → 높음)  
> Y축: 행동 실행도 (낮음 → 높음)

| | 목표 명확도 낮음 | 목표 명확도 높음 |
|---|---|---|
| **실행도 높음** | 방향탐색형 — 목표는 있으나 액션을 모른다 | 분산실행형 — 목표가 너무 많아 집중 못함 |
| **실행도 낮음** | 출발전단계 — 목표 자체가 없다 | 가시화필요형 — 결과가 잘 안 보인다 |

## 1차 인터뷰 타겟 페르소나 (확정)

### 페르소나: 성장기 팀의 중간 리더

> **"성장기 팀의 중간 리더로, 위에서 받은 방향을 팀에 어떻게 놓을지도 막막하고,  
> 팀원이 실제로 뭘 하고 있는지도 파악이 안 되고,  
> 목표가 시점이 지나면 흐려지는 경험을 반복하는 사람"**

#### 선정 이유

- 4가지 유형 중 **방향탐색형**을 우선 타겟으로 결정
  - 분산실행형은 너무 바쁨 → 새로운 툴 도입 여력 낮음
  - 출발전단계는 아직 Dowin이 필요한 시점이 아님
- **팀 단위**여야 한다 (개인 대상 아님)
- 팀 규모는 **성장기 팀 (10~50명 내외)**
  - 초기 혼돈 단계보다 방향은 잡혔으나 팀이 커지면서 정렬이 무너지는 단계
  - 반드시 회사 단위가 아니어도 됨 (사이드 프로젝트 팀 포함)
- **중간 리더 / 팀장** 역할이 핵심 대상
  - 위에서 OKR/방향을 받아 팀에 풀어야 하는 역할

#### 핵심 페인포인트 (3가지)

1. **팀원 개개인이 뭘 하고 있는지 파악이 안 된다**
2. **목표가 중간에 흐지부지되어 유명무실해진다**
3. **위에서 받은 방향을 팀 목표로 어떻게 쪼갤지 모른다**

#### 현재 흔한 관리 패턴 (주변에서 관찰된 사례)

- 노션/스프레드시트로 관리하는데 팀 전체가 안 쓰고 있음
- 비주기적인 회의로만 확인하고, 사이에 개인이 등한지는 모름
- 트레로/슬랙 등 소통 툴로만 메시지 위주로 돌아감 (목표가 주저앉음)
- 진행하고 있지만 막상 마일스톤을 가보면 스케줄이 깨져 있음
- 1:1로만 공유해서 시스템적으로 팀 전체 공유가 안 됨

## 인터뷰 설계 방향

### 인터뷰지 구성
- **2벌** 필요: 리더용 / 팀원용 (역할에 따라 질문 구조가 달라짐)
  - 리더용: "어떻게 팀을 정렬시킬까"
  - 팀원용: "목표가 어떻게 전달되고 있어? 실제로 체감이 돼?"

### 인터뷰 목표 우선순위
1. **페인포인트 발굴** — 얼마나 심각하게 느끼는지 (몸으로 부딪히는 상황)
2. **현재 툴 파악** — 경쟁제품 파악과 차별 포인트 확인

### 인터뷰 운영 방식
- 길이: 30분으로 시작, 관심 있으면 계속 이어간다
- 비IT 직군이 많으므로 전문 용어 최소화, 부담 없는 구조로 설계
